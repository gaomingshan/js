你是一名资深前端架构师与技术教育者，在 @html  下，请生成面向「后端开发者」的前端 HTML 系统化学习内容及面试题，要求如下：

【学习对象】
- 以 Java / Go / Node.js 等后端背景为主的开发者
- 对 HTML 不满足于“能写页面”，而是希望：
  - 从“文档系统”的角度理解 HTML
  - 建立类似后端领域模型、协议规范的结构化认知
  - 理解 HTML 在浏览器加载、解析、渲染链路中的真实作用

【学习目标】
- 建立 HTML 的整体技术地图，而非零散标签知识
- 从以下维度系统掌握 HTML：
  - 文档结构与语义模型
  - 标签系统与内容模型（Content Model）
  - 加载顺序、解析机制与执行时序
  - 渲染流程及与 CSS / JS 的协作关系
- 能在复杂前端工程（SPA / SSR / 多端）中理解 HTML 的职责边界

【生成顺序与文件结构】
1. 生成大纲文件 content-outline.md，每个小节带有 Markdown 跳转链接到对应 content 文件
2. 生成教学内容，存放于 content/ 目录
3. 生成面试题汇总，存放于 quiz/quiz.md

【文件命名规则】
- 大纲文件：content-outline.md
- 教学内容：content/content-1.md、content/content-2.md … 按大纲顺序递增
- 面试题汇总：quiz/quiz.md
- 链接格式：[小节标题](./content/content-x.md)，x 按大纲顺序递增
- 统一使用 Markdown，中文，技术严谨，面向工程理解

【教学内容要求】
- 内容必须以“体系化理解 HTML”为第一目标，而不是标签速查
- 核心内容必须围绕：
  - HTML 的文档模型（Document Model）与语义层级
  - HTML 标签的分类体系与内容模型约束
  - 浏览器对 HTML 的加载、解析与 DOM 构建流程
  - HTML 与 CSS / JavaScript 的执行顺序与协作机制
  - HTML 在渲染管线（Render Pipeline）中的位置
- 需要显式对标后端概念：
  - HTML 文档 ≈ 协议报文 / 数据结构
  - DOM ≈ 内存对象模型
  - 渲染流程 ≈ 请求处理管线

【每小节必须包含】
- 核心概念说明（是什么）
- 设计动机 / 工作原理（为什么 & 如何工作）
- 常见误区或边界问题（后端转前端高频踩坑）
- 工程实践示例（真实前端项目 / 服务端渲染场景）

【难度分级】
- 基础：
  - HTML 的角色定位、文档结构、基础语义
- 进阶：
  - 标签内容模型、加载与解析顺序、DOM 构建
- 高级：
  - 渲染机制、SSR / Hydration、HTML 与框架边界

【面试题要求】
- 单选 / 多选 / 代码分析 / 场景题结合，约 100 题
- 难度递进：基础认知 → 原理理解 → 架构级判断
- 覆盖范围必须与教学大纲完全一致
- 题目必须强调：
  - HTML 为什么要这样设计
  - 不同结构方案的取舍逻辑
  - HTML 与 CSS / JS / 框架的职责边界
- 每题提供答案与解析
- 解析格式要求：
  - 结论 → 原理 → 易错点 / 反例
  - 超过 15 行 Markdown 必须拆分
- 面试题仅生成一个汇总文件 quiz/quiz.md

【质量红线】
- 禁止标签堆砌式教学
- 禁止仅描述“怎么写”，不解释“为什么这样设计”
- 禁止脱离浏览器执行模型讲 HTML
- 禁止用前端黑话替代清晰抽象
- 概念不清时宁可删减，不可堆字

【额外说明】
- 明确对比 HTML 与后端常见结构化技术：
  - HTML vs JSON / XML
  - HTML 文档 vs HTTP Response Body
- 必须覆盖真实工程问题：
  - HTML 结构不合理导致的渲染与性能问题
  - JS 过度侵入 HTML 的风险
  - SSR / SPA 场景下 HTML 被弱化的问题
- 大纲文件必须生成 Markdown 链接，格式为 [小节标题](./content/content-x.md)，确保点击可跳转
