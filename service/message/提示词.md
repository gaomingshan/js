【教学内容要求】
- 核心思想：消息队列架构模型 + 一致性保证 + 性能优化优先
- 每小节包含：
  - 核心概念与设计动机
  - Kafka、RocketMQ、RabbitMQ 三者对比
  - 底层原理与实现机制
  - 易错点/边界条件

- 难度分级：基础 → 进阶 → 高级
- 教学内容和示例要覆盖真实企业场景
- 核心主题必须包含：

  **一、基础架构与模型**
  - 消息队列核心概念（Producer、Consumer、Broker、Topic、Partition、Queue）
  - 三大消息队列架构对比（Kafka、RocketMQ、RabbitMQ）
  - 消息模型（点对点、发布订阅、主题分区）
  - 存储模型（顺序写、页缓存、零拷贝）

  **二、环境搭建与部署**
  - 单机部署（快速体验）
  - 集群部署（生产级配置）
  - 高可用架构（主从、副本、故障转移）
  - 容器化部署（Docker、Kubernetes）
  - 监控与运维工具

  **三、核心机制**
  - Topic 管理（创建、删除、配置）
  - 分区机制（分区策略、动态扩容、负载均衡）
  - 副本机制（ISR、Leader 选举、数据同步）
  - 消息存储（持久化、清理策略、保留期限）
  - 索引机制（offset 索引、时间索引）

  **四、消息可靠性保证**
  - 消息发送可靠性（ACK 机制、重试策略、幂等性）
  - 消息存储可靠性（副本、刷盘策略、数据一致性）
  - 消息消费可靠性（消费确认、重平衡、位移管理）
  - 事务消息（本地事务、分布式事务、最终一致性）
  - 顺序消息（分区顺序、全局顺序）
  - 死信队列与重试机制

  **五、消费模型与进度管理**
  - 推模型 vs 拉模型（优劣对比、适用场景）
  - 消费者组与集群消费
  - 广播消费 vs 集群消费
  - 消费进度管理（offset 提交、自动提交、手动提交）
  - 消费位点重置（earliest、latest、指定位置）
  - 并发消费与顺序消费

  **六、性能优化**
  - 批量发送与批量消费
  - 压缩算法（GZIP、Snappy、LZ4、ZSTD）
  - 零拷贝与 mmap
  - 连接管理与连接池
  - 吞吐量优化（生产者、消费者、Broker）
  - 延迟优化（网络、磁盘、序列化）

  **七、配置详解**
  - 生产者配置（超时、重试、批量、压缩、幂等）
  - 消费者配置（拉取、提交、心跳、会话）
  - Broker 配置（网络、日志、副本、清理）
  - Topic 配置（分区数、副本数、保留期限、压缩策略）
  - 集群配置（Zookeeper、Controller、选举）

  **八、命令行管理**
  - Topic 管理（创建、查询、修改、删除、配置变更）
  - 分区管理（副本分配、重分配、优先副本选举）
  - 消息管理（查询、发送、消费、位移重置）
  - 消费者组管理（查询、描述、删除、位移管理）
  - 节点管理（上线、下线、迁移、负载均衡）
  - 集群管理（配置变更、动态配置、监控指标）

  **九、Spring 生态集成**
  - 原生 API 接入（Kafka Client、RocketMQ Client、RabbitMQ Java Client）
  - Spring 接入（Spring Kafka、Spring RocketMQ、Spring AMQP）
  - Spring Boot 接入（自动配置、注解驱动、配置文件）
  - Spring Cloud Stream（统一抽象、Binder 机制、多消息队列切换）
  - 事务消息与 Spring 事务整合
  - 消息序列化（JSON、Avro、Protobuf）

  **十、企业级实践**
  - 事件总线架构设计（事件定义、路由、版本管理）
  - 消息幂等性方案（业务幂等、去重表、分布式锁）
  - 消息顺序性保证（分区键设计、单线程消费）
  - 消息追踪与审计（链路追踪、日志收集）
  - 灰度发布与流量控制
  - 多集群部署与跨区域同步
  - 消息队列选型（业务场景、性能、成本）
  - 最佳实践与反模式

  **十一、运维与监控**
  - 监控指标（吞吐量、延迟、堆积、丢失）
  - 告警策略（消费延迟、集群异常、副本同步）
  - 故障排查（消息堆积、消费慢、重平衡频繁）
  - 容量规划（存储、网络、CPU、内存）
  - 备份与恢复策略
  - 升级与迁移方案
