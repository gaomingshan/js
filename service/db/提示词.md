【教学内容要求】
- 核心思想：SQL 语言模型 + 存储引擎 + 事务与并发控制优先
- 每小节包含：
  - 核心概念
  - 原理/设计动机
  - 易错点/边界
  - 生产实践示例（MySQL、Oracle、PostgreSQL 对比）
- 精简但完整，避免百科式罗列
- 难度分级：基础 → 进阶 → 高级
- 必须对比三大数据库的语法差异、特性差异、性能差异
- 教学内容和示例要覆盖常见生产环境实践场景
- 核心主题必须包含：

  **第一部分：SQL 语言基础**
  - DDL（数据定义语言）：CREATE、ALTER、DROP、TRUNCATE
  - DML（数据操作语言）：INSERT、UPDATE、DELETE、MERGE、UPSERT
  - DQL（数据查询语言）：SELECT、JOIN、子查询、窗口函数、CTE（公共表表达式）
  - DCL（数据控制语言）：GRANT、REVOKE、权限体系
  - TCL（事务控制语言）：COMMIT、ROLLBACK、SAVEPOINT
  - 用户管理与权限：创建用户、角色管理、授权体系、默认初始化脚本
  - 字符集与排序规则：UTF-8、GBK、排序规则（Collation）、大小写敏感

  **第二部分：数据库对象与存储**
  - 数据类型：数值、字符、日期、二进制、JSON/XML（三大数据库对比）
  - 表结构设计：主键、外键、唯一约束、检查约束、默认值
  - 索引机制：B+Tree、Hash、位图、全文索引、函数索引、部分索引、空间索引
  - 索引原理深入：
    - 索引数据结构：B+Tree 原理、叶子节点存储、非叶子节点存储
    - 聚簇索引 vs 非聚簇索引（InnoDB vs MyISAM）
    - 回表（Table Access by ROWID）、索引覆盖（Covering Index）
    - 联合索引与最左前缀原则（Leftmost Prefix Rule）
    - 索引下推（Index Condition Pushdown, ICP）
    - 索引合并（Index Merge）、索引排序优化
    - 索引选择性、基数统计、索引区分度
    - 索引失效场景：隐式转换、函数操作、like 前缀匹配、OR 条件
    - 索引碎片与重建（REBUILD vs REORGANIZE）
  - 视图、物化视图、分区表、临时表
  - 存储过程、函数、触发器、序列

  **第三部分：查询执行与优化**
  - SQL 执行流程：解析 → 优化 → 执行
  - 查询优化器：基于成本优化（CBO）、基于规则优化（RBO）
  - 执行计划分析：EXPLAIN、执行计划解读、成本估算、关键指标（rows、type、Extra）
  - 高级优化技术：
    - MRR（Multi-Range Read）：减少随机 I/O
    - BKA（Batched Key Access）：批量连接优化
    - 索引条件下推（ICP）应用场景
    - 松散索引扫描（Loose Index Scan）
  - 查询优化技巧：索引优化、JOIN 优化、子查询优化、分页优化、IN vs EXISTS
  - 统计信息：直方图、NDV、采样、统计信息更新策略、过期统计信息影响
  - Hint 提示：Oracle Hint、MySQL Optimizer Hint、PostgreSQL pg_hint_plan

  **第四部分：事务与并发控制**
  - ACID 特性：原子性、一致性、隔离性、持久性
  - 事务隔离级别：读未提交、读已提交、可重复读、串行化
  - 并发问题：脏读、不可重复读、幻读、丢失更新
  - 锁机制：表锁、行锁、意向锁、间隙锁、临键锁
  - 死锁：死锁检测、死锁预防、死锁恢复
  - MVCC（多版本并发控制）：实现原理、Undo/Redo、版本链、可见性判断
  - 乐观锁与悲观锁：应用场景、实现方式

  **第五部分：存储引擎与内部机制**
  - MySQL 存储引擎：InnoDB、MyISAM、Memory 对比
  - InnoDB 架构：缓冲池、Redo Log、Undo Log、Change Buffer
  - PostgreSQL 架构：MVCC 实现、VACUUM、自动清理
  - Oracle 架构：SGA、PGA、Redo Log、Undo Tablespace
  - WAL（Write-Ahead Logging）：日志先行、持久化保证
  - Checkpoint：检查点机制、脏页刷新、性能影响
  - 崩溃恢复：日志回放、数据恢复、一致性保证

  **第六部分：性能调优**
  - 硬件层优化：CPU、内存、磁盘 I/O、网络、RAID 配置
  - 配置参数优化：缓冲池大小、连接数、日志配置、并发控制、刷盘策略
  - SQL 调优：慢查询分析、索引优化、执行计划优化、查询改写
  - 表设计优化：范式与反范式、垂直拆分、水平拆分、冷热数据分离
  - 监控与诊断工具：
    - 慢查询日志（Slow Query Log）
    - Performance Schema / sys schema（MySQL）
    - AWR / ASH / ADDM（Oracle）
    - pg_stat_statements（PostgreSQL）
    - 等待事件分析、锁等待监控
  - 连接管理：连接池原理、连接复用、连接泄漏检测
  - 性能测试：基准测试、压力测试、TPS/QPS 测试、sysbench / pgbench

  **第七部分：高可用与分布式架构**
  - 主从复制：异步复制、半同步复制、同步复制（MGR）
  - 复制原理：Binlog（Row/Statement/Mixed）、GTID、复制延迟、复制过滤、并行复制
  - 读写分离：代理中间件（ProxySQL、MaxScale）、应用层实现、一致性问题、延迟处理
  - 高可用方案：MHA、MGR（MySQL Group Replication）、Oracle RAC、PostgreSQL Patroni/Repmgr
  - 分库分表深入：
    - 垂直拆分 vs 水平拆分
    - 分片策略：Range、Hash、一致性哈希、地理位置
    - 路由规则：分片键选择、多维分片
    - 全局表、广播表、ER 分片
    - 跨分片查询、分布式 JOIN
    - 分布式 ID 生成：雪花算法、数据库号段、UUID
  - 分布式事务：2PC、3PC、TCC、Saga、本地消息表、最终一致性
  - 数据迁移：全量迁移、增量同步、双写验证、灰度切换、回滚方案

  **第八部分：备份恢复与安全**
  - 备份策略：全量备份、增量备份、差异备份、备份周期
  - 备份工具：mysqldump、xtrabackup、pg_dump、RMAN
  - 恢复策略：完全恢复、时间点恢复（PITR）、表级恢复
  - 安全加固：权限最小化、网络隔离、SQL 注入防护、审计日志
  - 数据加密：透明数据加密（TDE）、传输加密、列加密

【三大数据库对比重点】
- **MySQL**：
  - InnoDB 存储引擎特性（行锁、MVCC、外键）
  - Binlog 复制、GTID、MGR
  - 分库分表生态（ShardingSphere、MyCAT）
  - 适用场景：互联网高并发、读写分离

- **Oracle**：
  - 企业级特性（RAC、Data Guard、Flashback）
  - 高级 SQL 特性（分析函数、层次查询）
  - PL/SQL 存储过程、包、游标
  - 适用场景：金融、电信等核心系统

- **PostgreSQL**：
  - 开源数据库中的"最先进"特性
  - 丰富的数据类型（JSON、数组、hstore、PostGIS）
  - 扩展性（插件机制、自定义类型/函数）
  - MVCC 实现、并发控制、逻辑复制
  - 适用场景：GIS、时序数据、复杂查询
