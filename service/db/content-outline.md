## 第二章：一条SQL的旅程 — 全景框架
> 对应文件：content-1（SQL完整旅程 + 执行器展开 + 存储引擎对比表）
**对日常有什么用**：以后看任何数据库问题，脑子都有完整的"请求路径"

- 一条SQL的一生：连接器 → 解析器 → 优化器 → 执行器 → 存储引擎
- 每个环节回答的问题：
  - 连接器：你是谁？（身份认证/权限）
  - 解析器：你想干嘛？（词法/语法/语义分析）
  - 优化器：怎么干最快？（成本计算，后文独立展开）
  - 执行器：按计划干活（Server层过滤/fiesort/LIMIT控制）
  - 存储引擎：真正读写数据
- SELECT 和 UPDATE 完整流程串起来
- 理解这个流程有什么用：看到慢查询能定位到具体环节

**存储引擎对比**：InnoDB vs MyISAM vs Memory vs 其他（速查表 + 选型建议）

## 第三章：索引 — 为什么快，怎么更快
> 对应文件：content-5（6子节）
**对日常有什么用**：90%的性能问题出在这里

### 正文（理解）：
- B+Tree 结构与查询路径（精确匹配/范围查询/页分裂）
- 聚簇索引 = InnoDB表本身
- 回表 = 随机IO（索引优化的核心矛盾）
- 覆盖索引 / ICP / MRR / Index Merge
- 联合索引与最左前缀（不是"规则"，是排序结构决定的）
- 失效场景（本质上都是破坏B+Tree有序性）

### 速查（参考）：
- B+Tree 查询路径详解
- 索引分类总表（13种类型 + 适用场景 + 支持数据库）
- 优化手段一览（7种手段 + 启用条件 + 优化优先级）
