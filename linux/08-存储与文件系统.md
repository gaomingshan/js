## 磁盘查看

### lsblk —— 查看磁盘/分区拓扑

```bash
lsblk               # 树形显示磁盘和分区关系
lsblk -f            # 显示文件系统类型和 UUID
lsblk -d -o name,size,rota  # rota=1 机械盘，=0 SSD
```

### fdisk / parted —— 分区表

```bash
fdisk -l            # MBR 分区表详情
parted -l           # GPT 分区（也支持 MBR）
```

### blkid —— UUID 和文件系统类型

```bash
blkid               # 所有块设备的 UUID + TYPE
blkid /dev/sdb1     # 指定设备
```

## 分区格式化

### 分区操作

```bash
fdisk /dev/sdb      # 交互式分区（n 新建 / w 写入 / p 显示 / q 退出）
parted /dev/sdb mklabel gpt   # 转 GPT
parted /dev/sdb mkpart primary ext4 1MiB 100%
```

### 格式化

```bash
mkfs.ext4 /dev/sdb1              # ext4
mkfs.xfs /dev/sdb2               # xfs
mkfs.btrfs /dev/sdb3             # btrfs
mkswap /dev/sdb4 && swapon /dev/sdb4  # swap
```

### 文件系统对比

| 特性 | ext4 | xfs | btrfs |
|------|------|-----|-------|
| 最大文件 | 16TB | 8EB | 16EB |
| 最大卷 | 1EB | 8EB | 16EB |
| 在线扩容 | ✓ | ✓ | ✓ |
| 快照 | ✗ | ✗ | ✓ |
| 压缩 | ✗ | ✗ | ✓ |
| 适用 | 通用 | 大文件/高性能 | 高级功能 |

## LVM

### 创建 LVM

```bash
pvcreate /dev/sdb                  # 创建物理卷
vgcreate vg_data /dev/sdb         # 创建卷组
lvcreate -L 50G -n lv_data vg_data  # 创建逻辑卷
mkfs.ext4 /dev/vg_data/lv_data    # 格式化
```

### 查看 LVM

```bash
pvdisplay / pvs                   # 物理卷
vgdisplay / vgs                   # 卷组
lvdisplay / lvs                   # 逻辑卷
```

### 扩容

```bash
# 卷组有空间 → 直接扩
lvextend -L +50G /dev/vg_data/lv_data
lvextend -l +100%FREE /dev/vg_data/lv_data  # 全部分配

# 文件系统感知新大小
resize2fs /dev/vg_data/lv_data    # ext4
xfs_growfs /mount/point            # xfs（必须挂载点）
```

### 缩容（ext4 仅支持离线）

```bash
# ext4 缩容必须先卸载
umount /data
e2fsck -f /dev/vg_data/lv_data
resize2fs /dev/vg_data/lv_data 400G
lvreduce -L 400G /dev/vg_data/lv_data
mount /dev/vg_data/lv_data /data
```

### 快照

```bash
lvcreate -s -L 10G -n snap_pre_update /dev/vg_data/lv_data
# -s 快照，-L 给快照分配的空间（不是原始大小）

# 回滚
lvconvert --merge /dev/vg_data/snap_pre_update

# 删除快照
lvremove /dev/vg_data/snap_pre_update
```

## 挂载

### mount / umount

```bash
mount /dev/sdb1 /data                  # 挂载设备
mount -o ro /dev/sdb1 /data            # 只读挂载
mount -o remount,rw /data              # 重新挂载为读写
mount -o noexec /data                  # 禁止执行二进制
mount --bind /a /b                     # 目录绑定

# NFS 挂载
mount -t nfs 10.0.0.1:/export /mnt/nfs -o nfsvers=4.2,rsize=1048576

umount /data                           # 卸载
umount -l /data                        # 懒卸载（等无人使用时再卸）
```

### fstab —— 开机挂载

```bash
# <设备>        <挂载点>  <类型>  <选项>         <dump> <pass>
UUID=xxx        /data     ext4    defaults,noatime 0      2
/dev/vg_data/lv_data  /app  xfs  defaults,noatime  0      2
10.0.0.1:/export      /nfs  nfs   defaults,_netdev 0      0
```

常见选项：

| 选项 | 含义 |
|------|------|
| `defaults` | rw, suid, dev, exec, auto, nouser, async |
| `noatime` | 不更新访问时间，减少 IO |
| `nodiratime` | 不更新目录访问时间 |
| `_netdev` | 网络设备，网络就绪后才挂载 |
| `noexec` | 禁止执行脚本/二进制 |

## 容量排查

### df —— 文件系统使用

```bash
df -h                 # 人类可读大小
df -i                 # inode 使用（inode 满了也能导致写入失败）
df -T                 # 显示文件系统类型
```

### du —— 目录大小

```bash
du -sh *              # 当前目录下各项大小
du -ch --max-depth=1 / | sort -rh   # 一级目录从大到小
du -sh /var/log       # 指定目录总量
du -sh --exclude=proc /  # 排除 proc
```

### ncdu —— 交互式磁盘分析

```bash
ncdu /                # 交互式浏览，支持排序/删除（需安装）
```

### 排查大文件

```bash
# 超过 1GB 的文件
find / -type f -size +1G -exec ls -lh {} \; 2>/dev/null

# 当前目录最大 10 个文件
find . -type f -exec du -Sh {} + | sort -rh | head -10
```

## 性能排查

### iostat —— 磁盘 IOPS + 延迟

```bash
iostat -x 1           # 每秒输出，含 rrqm/s wrqm/s rkB/s wkB/s await svctm %util
iostat -x 1 5         # 每秒一次，共 5 次
```

关键指标：

| 指标 | 含义 | 警戒值 |
|------|------|--------|
| `%util` | 磁盘繁忙比 | > 80% |
| `await` | IO 平均等待时间（ms） | > 30ms 机械盘 |
| `svctm` | IO 平均服务时间（ms） | 越接近 await 说明越健康 |
| `r_await / w_await` | 读写分离等待 | 分别看 |
| `aqu-sz` | 平均队列长度 | > 1 |

### iotop —— 按进程看 IO

```bash
iotop                 # 交互，按 O 排序
iotop -o              # 只显示有 IO 的进程
iotop -bk             # 批模式，适合脚本
```

### dstat —— 汇总

```bash
dstat -d              # 磁盘读写汇总
dstat -d --top-io     # 磁盘 + IO 最高进程
dstat -d --top-bio    # 磁盘 + 块 IO 最高进程
```

### sar —— 历史统计

```bash
sar -d 1 10           # 当前磁盘统计
sar -d -f /var/log/sa/sa01  # 查看某天的历史
sar -b 1 3            # IO 请求 + 传输速率
```

## NFS

### 服务端

```bash
# 安装
apt install nfs-kernel-server   # Debian/Ubuntu
yum install nfs-utils           # CentOS/RHEL

# 配置导出
echo "/data  *(rw,sync,no_subtree_check,no_root_squash)" >> /etc/exports
exportfs -a           # 全部导出
exportfs -v           # 查看导出列表
exportfs -ra          # 重新导出
```

`/etc/exports` 选项：

| 选项 | 含义 |
|------|------|
| `rw` | 读写 |
| `sync` | 同步写入（推荐） |
| `async` | 异步写入（性能好但不安全） |
| `no_root_squash` | 客户端 root 保留 root 权限 |
| `no_subtree_check` | 禁用子树检查（性能） |
| `insecure` | 允许 > 1024 端口连接 |

### 客户端

```bash
mount -t nfs server:/data /mnt/nfs
mount -t nfs server:/data /mnt/nfs -o rsize=1048576,wsize=1048576,hard,intr

# fstab
server:/data  /mnt/nfs  nfs  defaults,_netdev  0  0
```

NFS 挂载选项：

| 选项 | 含义 |
|------|------|
| `hard` | 服务端挂起则一直重试（推荐） |
| `soft` | 超时返回错误，避免进程挂死 |
| `intr` | 允许中断挂起的 IO |
| `rsize=1048576` | 读块大小 1MB |
| `wsize=1048576` | 写块大小 1MB |

### NFS 性能排查

```bash
nfsstat -s            # 服务端统计
nfsstat -c            # 客户端统计
mountstat             # NFS 挂载统计
showmount -e server   # 查看服务端导出
```

## 实战场景

### 场景 1：日志盘写满紧急处理

```bash
# 1. 定位
df -h
du -sh /var/log/* | sort -rh | head -10

# 2. 清理（先大文件，再老文件）
find /var/log -name "*.log" -mtime +7 -delete

# 3. 如果日志进程持有已删文件句柄 → 空间不释放
lsof | grep "(deleted)"    # 查找已删还在占用的
> /var/log/access.log      # 或 kill -HUP 日志进程

# 4. 紧急转移
mkdir /data/log && rsync -av /var/log/ /data/log/
mount --bind /data/log /var/log
```

### 场景 2：云盘在线扩容（不重启）

```bash
# 1. 云控制台扩盘 → OS 感知新大小
echo 1 > /sys/block/sda/device/rescan   # 或重启
lsblk                                     # 确认新大小

# 2. 分区扩容（最后一块分区）
growpart /dev/sda 1
# 或直接用 parted 调整

# 3. 文件系统扩容
resize2fs /dev/sda1    # ext4
xfs_growfs /mount/point  # xfs（需要挂载）

# 一键脚本（ext4）
growpart /dev/sda 1 && resize2fs /dev/sda1
```

### 场景 3：LVM 快照备份数据库

```bash
# 1. 创建快照
lvcreate -s -L 20G -n snap_mysql /dev/vg_data/lv_mysql

# 2. 挂载快照 → 备份
mkdir /mnt/snap
mount /dev/vg_data/snap_mysql /mnt/snap -o ro
mysqldump --all-databases > /backup/db.sql   # 或 cp -a 整个数据目录

# 3. 清理
umount /mnt/snap
lvremove /dev/vg_data/snap_mysql
```

### 场景 4：找到占用 IO 最高的进程

```bash
# 方法一：iotop
iotop -o

# 方法二：/proc 手动查
for pid in /proc/[0-9]*; do
  read=$(cat $pid/io 2>/dev/null | grep read_bytes | awk '{print $2}')
  write=$(cat $pid/io 2>/dev/null | grep write_bytes | awk '{print $2}')
  echo "$(basename $pid) r=$read w=$write"
done | sort -t= -k3 -rn | head -10

# 方法三：dstat + top
dstat -d --top-io 5
```

### 场景 5：NFS 性能调优

```bash
# 客户端调大读写块
mount -o rsize=1048576,wsize=1048576,noatime,hard,intr ...

# 加大 NFS 内核线程
echo "options nfs nfs_callback_tcpport=32765" > /etc/modprobe.d/nfs.conf

# 服务端调优
echo 262144 > /proc/sys/net/core/rmem_default
echo 262144 > /proc/sys/net/core/wmem_default
```
