# 压缩与归档

> 文件操作 | tar/zip/gzip | 日志归档、备份传输、磁盘空间管理

---

## tar —— 打包专用（不压缩只打包 / 配合压缩算法）

```bash
tar -cf archive.tar dir/                     # 只打包，不压缩
tar -czf archive.tar.gz dir/                 # 打包 + gzip（最常用）
tar -cjf archive.tar.bz2 dir/                # bzip2（压缩率更高）
tar -cJf archive.tar.xz dir/                 # xz（压缩率最高，最慢）

tar -xf archive.tar.gz                       # 解压
tar -xf archive.tar.gz -C /target/dir        # 解压到指定目录
tar -tf archive.tar.gz                       # 不解压查看内容
```

### 高级用法

```bash
# 排除文件/目录
tar -czf app.tar.gz /app --exclude='*.log' --exclude='node_modules'

# 带日期命名
tar -czf backup_$(date +%Y%m%d_%H%M%S).tar.gz /data

# 增量打包 —— 只打包某天后修改的文件
tar -czf incremental.tar.gz $(find /data -type f -mtime -1)

# 解压特定文件
tar -xf archive.tar.gz file1.txt dir/file2.py

# 追加到已有 tar（仅限非压缩包）
tar -rf archive.tar newfile.txt

# 远程传输：本地打包直接发送到远程解压
tar -czf - /app | ssh user@host "tar -xzf - -C /opt"
```

### 实用技巧

```bash
# 查看压缩包内容（不解压）
tar -tf archive.tar.gz

# 查看详细信息（权限、大小、时间）
tar -tvf archive.tar.gz

# 验证包完整性
tar -tzf archive.tar.gz > /dev/null && echo "OK" || echo "Corrupt"
```

## 压缩算法对比

| 格式 | 命令选项 | 压缩率 | 速度 | 常用场景 |
|------|---------|--------|------|---------|
| `.tar.gz` | `z` | 中 | 快 | 日常打包、软件分发（最常用） |
| `.tar.bz2` | `j` | 高 | 中 | 大文件传递、CI 产物 |
| `.tar.xz` | `J` | 最高 | 慢 | 归档冷数据、备份保存 |
| `.tar.zst` | `--zstd` | 高 | 快 | 现代 Linux 新宠（平衡） |

速度对比示意（同文件）：
```
gzip  ≈ 1x 速度, 1x  大小
bzip2 ≈ 2x 速度, 0.7x 大小
xz    ≈ 5x 速度, 0.5x 大小
zstd  ≈ 1.5x 速度, 0.6x 大小
```

## gzip / bzip2 / xz —— 单文件压缩

```bash
gzip file.log                              # 压缩 → file.log.gz（原文件消失）
gzip -k file.log                           # 保留原文件（-k = keep）
gzip -d file.log.gz                        # 解压 → file.log
gunzip file.log.gz                         # 同上
gzip -l file.log.gz                        # 查看压缩率

bzip2 file.log                             # → file.log.bz2
bunzip2 file.log.bz2                       # 解压

xz file.log                                # → file.log.xz
unxz file.log.xz                           # 解压
```

### 配合 find 批量压缩

```bash
# 压缩 7 天前的日志
find /var/log -name "*.log" -mtime +7 -exec gzip {} \;

# 解压所有 .gz
find /var/log -name "*.gz" -exec gunzip {} \;

# 压缩并移动到归档目录
find /var/log -name "*.log" -mtime +7 -exec gzip {} \; -exec mv {}.gz /archive/
```

## zip / unzip —— 跨平台压缩

Windows 原生支持，适合与同事交换文件。

```bash
zip -r archive.zip dir/                     # 递归压缩目录
zip archive.zip file1 file2                 # 压缩多个文件
zip -q archive.zip dir/                     # 静默模式
zip -P password archive.zip dir/            # 加密（⚠️ 密码在 ps 里可见）

unzip archive.zip                           # 解压
unzip -l archive.zip                        # 查看内容（不解压）
unzip -d /target archive.zip                # 解压到指定目录
unzip -o archive.zip                        # 覆盖已存在的文件

# 只解压特定文件
unzip archive.zip "*.txt"
```

## 分卷压缩

大文件要刻盘或受上传大小限制时需要分卷。

```bash
# 每卷 100MB（split 分卷 + tar 打包合并）
tar -czf - large_dir/ | split -b 100M - archive.tar.gz.

# 合并解压
cat archive.tar.gz.* | tar -xzf -

# 直接分卷 zip
zip -s 100m -r archive.zip large_dir/
```

## 加密压缩

```bash
# tar + openssl 加密
tar -czf - sensitive_dir/ \
  | openssl enc -aes-256-cbc -salt -out archive.tar.gz.enc

# 解密
openssl enc -d -aes-256-cbc -in archive.tar.gz.enc \
  | tar -xzf -
```

## 实践场景

### 场景 1：备份数据库导出文件

```bash
# 导出 MySQL + 压缩 + 日期命名
mysqldump -u root mydb | gzip > backup_mydb_$(date +%Y%m%d).sql.gz

# 批量备份多个数据库
mysql -N -e "SHOW DATABASES" | grep -v "information_schema\|performance_schema" \
  | while read db; do
      mysqldump -u root "$db" | gzip > "backup_${db}_$(date +%Y%m%d).sql.gz"
    done
```

### 场景 2：日志轮转归档

```bash
# 每日凌晨压缩前一天的日志
find /var/log/myapp -name "*.log" -mtime 0 -exec gzip {} \;

# 30 天前的日志打包后删除源文件
find /var/log/myapp -name "*.log.gz" -mtime +30 \
  | xargs tar -czf old_logs_$(date +%Y%m).tar.gz

# 删掉 90 天前的归档包
find /archive -name "*.tar.gz" -mtime +90 -delete
```

### 场景 3：跨服务器传输大目录

```bash
# 打包 → 压缩 → 传输 → 解压（一条命令）
tar -czf - /data/project/ \
  | pv \
  | ssh user@backup-server "cat > /backup/project_$(date +%Y%m%d).tar.gz"

# pv 会显示传输进度和速度
```

### 场景 4：分卷上传到对象存储

```bash
# 50M 分卷（S3/MinIO 单文件大小限制）
tar -czf - big_dataset/ | split -b 50M - dataset.tar.gz.

# 上传每卷
for f in dataset.tar.gz.*; do
  aws s3 cp "$f" s3://my-bucket/dataset/
done

# 下载合并
aws s3 sync s3://my-bucket/dataset/ ./
cat dataset.tar.gz.* | tar -xzf -
```

### 场景 5：部署包发布

```bash
# 构建产物打包，排除开发依赖
tar -czf release-$(git describe --tags).tar.gz \
  --exclude='node_modules' \
  --exclude='.git' \
  --exclude='test' \
  --exclude='*.map' \
  dist/ package.json yarn.lock

# 部署到服务器
scp release-*.tar.gz deploy@host:/app/
ssh deploy@host "cd /app && tar -xzf release-*.tar.gz && ln -snf release-* current"
```
