# 综合排障实战

> 全书收尾 | 故障现象→排查链路→根因定位 | 把前面命令串成完整流程

---

## 故障 1：CPU 100%

```
现象：服务变慢，top 显示 CPU 100%
```

**排查链路：**

```bash
# 1. top 定位高 CPU 进程
top

# 2. 看进程内哪个线程占 CPU
top -H -p PID

# 3. 把线程 ID 转十六进制（用于匹配 strace/perf）
printf "%x\n" TID

# 4. 统计系统调用耗时
strace -p PID -c

# 5. 只看网络相关的系统调用
strace -p PID -e trace=network

# 6. 热点函数采样
perf top -p PID

# 7. 看内核栈（是否卡在某个内核函数里）
cat /proc/PID/stack

# 8. 看用户态栈
# Java 进程
jstack PID
# C/C++ 进程
pstack PID
gdb -batch -ex "thread apply all bt" -p PID
```

**常见根因：**
- 死循环（while 忘记 sleep）
- 频繁 GC（Java Full GC 飙高）
- 大量正则回溯
- 内核态高（如磁盘 IO 等待、网络软中断）

## 故障 2：内存持续增长

```
现象：RES 缓慢上升，不回落，最终 OOM
```

**排查链路：**

```bash
# 1. 按内存排序找进程
top -o %MEM

# 2. 查看进程内存映射，哪个段最大
pmap -x PID | sort -k3 -rn | head

# 3. 更详细的内存段（PSS 更准确）
cat /proc/PID/smaps | grep -A5 "PSS:"

# 4. dump 内存后分析
gcore -o /tmp/core PID

# 5. Java 堆分析
jmap -heap PID                # 堆配置概览
jmap -histo:live PID | head   # 存活对象统计
jmap -dump:live,file=/tmp/heap.hprof PID  # dump 堆
```

**常见根因：**
- Java：ThreadLocal 未清理、静态集合无限增长、String.intern() 滥用
- C/C++：malloc/free 不匹配、内存池未释放
- 缓存层（Redis/Guava Cache）配置不合理

## 故障 3：磁盘 IO 打满

```
现象：iostat 显示 %util 接近 100%，应用读写慢
```

**排查链路：**

```bash
# 1. 确认哪块盘 IO 高
iostat -x 1

# 2. 找高 IO 的进程
iotop

# 3. 查看进程打开了哪些文件
lsof -p PID | grep REG | head

# 4. 追踪读写系统调用
strace -e trace=read,write -p PID 2>&1 | head -50

# 5. 看历史趋势
sar -d 1 10
```

**常见根因：**
- 日志写太猛（关 debug 日志、异步日志）
- 数据库大量全表扫描
- Swap 抖动（内存不足→换入换出→IO 打满→恶性循环）

## 故障 4：网络丢包

```
现象：ping 丢包、TCP 重传率高、连接超时
```

**排查链路：**

```bash
# 1. 找丢包在哪一跳
mtr host

# 2. 网卡硬件错误
ethtool -S eth0 | grep error

# 3. 内核丢包统计
ip -s link show eth0

# 4. 抓包分析
tcpdump -i eth0 -w /tmp/capture.pcap
# 然后用 Wireshark 分析

# 5. TCP 连接状态 + 重传
ss -ti

# 6. 查看连接队列溢出
netstat -s | grep -i "overflow\|drop"

# 7. 查看 socket 缓冲区
ss -t -p -i
```

**常见根因：**
- 带宽打满（查看 /proc/net/dev）
- 防火墙/iptables 丢包
- 连接队列满（listen backlog 太小）
- MTU 不匹配
- 网卡/网线故障

## 故障 5：进程 D 状态（不可中断睡眠）

```
现象：ps aux 显示 STAT=D，kill -9 杀不掉
```

**排查链路：**

```bash
# 1. 找到 D 状态进程
ps aux | grep " D"

# 2. 看内核在等什么
cat /proc/PID/stack

# 3. 看存储报错
dmesg | tail -50

# 4. 检查挂载状态
mount | grep nfs

# 5. 检查 NFS 服务器连通性
showmount -e nfs-server
```

**解决方案：**
- D 状态是内核在等待 IO 完成，**无法被杀死**
- 如果存储恢复，进程自动退出 D 状态
- 如果存储死透了，只能重启机器
- NFS 挂载超时可以用 `soft,intr` 挂载选项避免 D 状态

## 故障 6：文件句柄耗尽

```
现象：系统报 "Too many open files"
```

**排查链路：**

```bash
# 1. 用户级限制
ulimit -n

# 2. 系统级上限
cat /proc/sys/fs/file-max

# 3. 当前已用
cat /proc/sys/fs/file-nr
# 输出：已分配  已使用(但未分配)  上限

# 4. 某进程打开了多少句柄
lsof -p PID | wc -l

# 5. 都是些什么文件
lsof -p PID | head -30

# 6. 直接看 fd 目录
ls -l /proc/PID/fd | head

# 7. 修改限制
# /etc/security/limits.conf
* soft nofile 655360
* hard nofile 655360
```

**常见根因：**
- 连接未关闭（数据库连接池泄漏）
- 日志文件未轮转，每个 worker 都拿着 fd
- inotify 数量超限（`cat /proc/sys/fs/inotify/max_user_watches`）

## 故障 7：端口冲突

```
现象：服务报 "Address already in use"
```

**排查链路：**

```bash
# 1. 查端口被谁占
ss -tlnp | grep :8080
netstat -tlnp | grep 8080

# 2. 杀占用进程
fuser -k 8080/tcp

# 3. 或者换端口
# 改应用配置文件，避开冲突端口

# 4. 如果是 TIME_WAIT 过多
sysctl -w net.ipv4.tcp_tw_reuse=1
sysctl -w net.ipv4.tcp_fin_timeout=30
```

## 故障 8：启动慢

```
现象：机器启动到服务可用耗时太长
```

**排查链路：**

```bash
# 1. 总启动时间
systemd-analyze

# 2. 每个服务的耗时排序
systemd-analyze blame

# 3. 关键启动路径
systemd-analyze critical-chain

# 4. 找到耗时大户后针对性优化
# 常见：network.target 等待 DHCP 超时
# 解决：对不依赖网络的服务设 After=network-online.target
```

## 实战：新服务器初始化清单

```bash
# 1. 主机名
hostnamectl set-hostname app-prod-01

# 2. 时区
timedatectl set-timezone Asia/Shanghai

# 3. 镜像源
# Debian/Ubuntu: sed -i 's/deb.debian.org/mirrors.ustc.edu.cn/g' /etc/apt/sources.list
# CentOS/RHEL: sed -i 's/mirrorlist/#mirrorlist/; s/baseurl=/baseurl=http:\/\/mirrors.aliyun.com\//' /etc/yum.repos.d/*.repo

# 4. SSH 加固
sed -i 's/^#Port 22/Port 2222/;
        s/^PermitRootLogin yes/PermitRootLogin no/;
        s/^PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config
systemctl restart sshd

# 5. 内核参数优化
cat >> /etc/sysctl.conf <<'EOF'
net.ipv4.tcp_syncookies=1
net.ipv4.tcp_tw_reuse=1
net.core.somaxconn=65535
vm.swappiness=10
fs.file-max=1000000
net.ipv4.tcp_fin_timeout=30
EOF
sysctl -p

# 6. 文件句柄限制
cat >> /etc/security/limits.conf <<'EOF'
* soft nofile 655360
* hard nofile 655360
EOF

# 7. NTP 时间同步
systemctl enable --now chronyd
# 或
timedatectl set-ntp true

# 8. 关闭不用的服务
systemctl disable --now postfix  # 邮件服务（服务器通常不用）

# 9. 创建统一部署用户
useradd -m -s /bin/bash deploy
echo "deploy ALL=(ALL) NOPASSWD:ALL" >> /etc/sudoers.d/deploy

# 10. 防火墙
systemctl enable --now firewalld
firewall-cmd --add-service=ssh --permanent
firewall-cmd --add-port=2222/tcp --permanent
# 只开放需要的端口
```

## 实战：日常巡检脚本

```bash
#!/bin/bash
echo "========================================"
echo "  服务器巡检报告 - $(hostname)"
echo "  时间: $(date)"
echo "========================================"

echo ""
echo "=== CPU 负载 ==="
uptime
echo ""
echo "=== 内存 ==="
free -h
echo ""
echo "=== 磁盘 ==="
df -h | grep -v tmpfs | grep -v devtmpfs
echo ""
echo "=== 网络连接数 ==="
ss -tan | awk '{print $1}' | sort | uniq -c | sort -rn
echo ""
echo "=== 监听端口 ==="
ss -tlnp4 | awk 'NR>1 {print $4, $7}'
echo ""
echo "=== 关键进程 ==="
for proc in nginx mysqld java redis-server postgresql; do
  if pgrep -x "$proc" > /dev/null 2>&1; then
    echo "  [OK]    $proc"
  else
    echo "  [DOWN]  $proc"
  fi
done
echo ""
echo "=== 登录失败记录（最近 5 条） ==="
lastb -n 5 2>/dev/null || echo "  无失败登录"
echo ""
echo "========================================"
```
