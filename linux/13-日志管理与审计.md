# 日志管理与审计

> 查看/轮转/转发日志 | journalctl/rsyslog/logrotate/sysstat | 问题排查与监控

---

## 基本查看（与第一章衔接）

```bash
tail -f /var/log/nginx/access.log          # 实时跟踪请求
tail -F /var/log/nginx/access.log          # 自动重跟轮转后的文件
less /var/log/syslog                       # 翻页查看
zgrep "ERROR" /var/log/nginx/error.log.1.gz  # 搜索压缩日志
```

## journalctl（systemd 日志）

systemd 统一收集所有服务的日志，替代散落在 `/var/log/` 的零散文件。

```bash
journalctl -u nginx -f                     # 实时看某服务日志
journalctl --since "1 hour ago" -u nginx   # 近 1 小时
journalctl --since "2026-06-08 10:00" --until "2026-06-08 12:00"  # 时间范围
journalctl -x -u nginx                     # -x 附错误解释（方便排查）
journalctl _PID=1234                       # 按进程 ID 过滤
journalctl -p err                          # 按日志级别（emerg/alert/crit/err/warning/info/debug）
```

### 日志空间管理

```bash
journalctl --disk-usage                    # 占了多少空间
journalctl --vacuum-size=200M              # 限制最大 200M
journalctl --vacuum-time=7d                # 保留 7 天
journalctl --vacuum-files=5                # 保留 5 个文件

# 配置文件 /etc/systemd/journald.conf
# SystemMaxUse=200M
# MaxRetentionSec=7day
```

### 持久化

默认日志在内存中，重启丢失。持久化需建目录：

```bash
mkdir -p /var/log/journal
systemd-tmpfiles --create --prefix /var/log/journal
systemctl restart systemd-journald
```

## rsyslog

传统 syslog 守护进程，支持过滤、转发、自定义日志文件。

```bash
# 转发到中央日志服务器
# /etc/rsyslog.conf
*.* @10.0.0.100:514          # UDP 转发
*.* @@10.0.0.100:514         # TCP 转发（更可靠）

# 自定义程序日志单独存文件
# /etc/rsyslog.d/myapp.conf
if $programname == "myapp" then /var/log/myapp.log
& stop                         # 不再往下写
```

| 协议 | 语法 | 特点 |
|------|------|------|
| UDP | `@host:port` | 快但不保证送达 |
| TCP | `@@host:port` | 可靠但有延迟 |
| RELP | `:omrelp:` | 可靠 + 应用层确认 |

## logrotate

日志轮转的标准工具，防止日志无限膨胀撑爆磁盘。

```bash
# /etc/logrotate.d/nginx
/var/log/nginx/*.log {
    daily                     # 每天轮转
    rotate 30                 # 保留 30 份
    compress                  # 压缩旧日志
    delaycompress             # 延迟一次（刚轮转的不压缩）
    missingok                 # 文件不存在不报错
    notifempty                # 空文件不轮转
    copytruncate              # 复制后截断原文件（Nginx 适用，不需要 reload）
    postrotate
        systemctl reload nginx > /dev/null 2>&1
    endscript
}
```

### 手动测试

```bash
logrotate -d /etc/logrotate.d/nginx        # debug 模式（模拟）
logrotate -f /etc/logrotate.d/nginx        # 强制执行
logrotate -f /etc/logrotate.d/nginx -s /tmp/logrotate.status  # 指定状态文件
```

### 常见轮转策略

| 频率 | 适用场景 | 保留 |
|------|---------|------|
| `daily` + `rotate 30` | 日常业务日志 | 30 天 |
| `hourly` + `rotate 168` | 高流量日志 | 7 天 |
| `size 100M` | 按大小 | 最近 N 个 |
| `monthly` + `rotate 12` | 低频审计日志 | 1 年 |

## sysstat 性能工具集

`sysstat` 包包含 `sar`、`iostat`、`mpstat`、`pidstat`，按固定间隔采集系统指标，事后回溯分析。

### sar 常用

```bash
sar -u 1 10                   # CPU 使用率，每秒一次，10 次
sar -r 1 10                   # 内存使用
sar -b 1 10                   # IO 吞吐
sar -n DEV 1 10               # 网络流量（每网卡）
sar -S 1 10                   # Swap 使用
sar -d 1 10                   # 磁盘 IO 统计
sar -q 1 10                   # 负载队列长度
sar -W 1 10                   # 每秒换页数
```

### 回溯历史

```bash
sar -u -f /var/log/sysstat/sa09    # 看 9 号的历史数据
sar -u -s 14:00:00 -e 14:30:00     # 看某时间范围
sar -P 0 1 10                      # 只看 CPU 0（多核排查）
```

### 其他工具

```bash
iostat -x 1                        # 详细 IO（await/svctm/util%）
mpstat -P ALL 1                    # 每颗 CPU 详细占用
pidstat -l 1                       # 按进程看资源
pidstat -p 1234 -r 1               # 看某进程内存
```

## 实践场景

### 场景 1：应用日志集中转发到 ELK

```bash
# 客户端（Filebeat 或 rsyslog）
# /etc/rsyslog.d/elk.conf
template(name="jsonTemplate" type="list") {
    property(name="timereported" dateFormat="rfc3339")
    property(name="hostname")
    property(name="programname")
    property(name="msg")
}
*.* @10.0.0.100:5514;jsonTemplate

# 服务端（Logstash）接收后送 Elasticsearch
```

### 场景 2：logrotate 不生效排查

```bash
# 1. 检查最后一次执行时间
cat /var/lib/logrotate/status | grep nginx

# 2. 手动 debug 模拟
logrotate -d /etc/logrotate.d/nginx

# 3. 强制执行一次
logrotate -f /etc/logrotate.d/nginx

# 4. 检查 cron 是否在跑
ls -la /etc/cron.daily/logrotate

# 5. 常见问题：文件权限不对、postrotate 脚本失败、状态文件脏了
rm -f /var/lib/logrotate/status && logrotate -f /etc/logrotate.conf
```

### 场景 3：历史命令与用户审计

```bash
# 所有用户的 bash 历史（默认只记录当前用户）
cat /var/log/auth.log | grep "session opened"        # 登录记录
last                                                     # 最近登录
lastlog                                                  # 所有用户上次登录
w                                                        # 当前在线用户

# 启用详细 auditd（受控环境）
auditctl -w /etc/passwd -p wa -k passwd_change          # 监控 passwd 修改
ausearch -k passwd_change                                # 查询审计记录

# 历史命令增强（追加到 /etc/bash.bashrc）
# export HISTTIMEFORMAT="%F %T "
# export HISTSIZE=10000
# export HISTFILE=/var/log/.bash_history
# readonly HISTFILE
# PROMPT_COMMAND="history -a; $PROMPT_COMMAND"  # 实时写入
```

### 场景 4：用 sysstat 排查昨天 14:00 的 CPU 飙高

```bash
# 1. 找到昨天的数据文件
ls -la /var/log/sysstat/
# 假设今天 9 号，昨天的文件是 sa08

# 2. 看 14:00~14:30 的 CPU
sar -u -f /var/log/sysstat/sa08 -s 14:00:00 -e 14:30:00

# 3. 定位是哪个进程
# 如果开了 pidstat 记录：
pidstat -l -f /var/log/sysstat/sa08 -s 14:00:00 -e 14:30:00 | head

# 4. 结合应用日志交叉确认
zgrep "14:" /var/log/myapp/error.log.1.gz | head -20
```
