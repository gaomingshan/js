# 包管理

> 安装/升级/删除软件 | apt/yum/dpkg/rpm/源码 | 系统维护基础

---

## apt（Debian/Ubuntu）

```bash
apt update                    # 刷新仓库索引
apt upgrade                   # 升级所有包（不删已装）
apt full-upgrade              # 升级可能删旧包
apt install nginx             # 安装（自动处理依赖）
apt remove nginx              # 卸载，保留配置文件
apt purge nginx               # 卸载 + 删配置
apt autoremove                # 清理不再需要的自动依赖
apt list --installed          # 列出已安装
apt list --upgradable         # 可升级的
apt-cache search keyword      # 搜索包名/描述
apt-cache show nginx          # 查看包详细信息
```

## yum/dnf（CentOS/RHEL 8+）

```bash
yum install -y nginx          # 安装（-y 静默确认）
yum remove nginx              # 卸载
yum list installed            # 已装列表
yum list available            # 仓库可用包
yum provides */netstat        # 查找文件属于哪个包
yum history                   # yum 操作历史（可 undo）
yum info nginx                # 包详情

# dnf 是 yum 的下一代，语法兼容
dnf groupinstall "Development Tools"  # 安装组包
```

| 功能 | apt | yum/dnf |
|------|-----|---------|
| 刷新索引 | `apt update` | `yum makecache` |
| 安装 | `apt install` | `yum install` |
| 搜索 | `apt-cache search` | `yum search` |
| 包信息 | `apt-cache show` | `yum info` |
| 文件归属 | `dpkg -S` | `yum provides` |

## dpkg / rpm（底层包工具）

```bash
# dpkg（Debian 系）
dpkg -i package.deb           # 安装（可能缺依赖报错）
dpkg -l | grep nginx          # 列出已装
dpkg -L nginx-common          # 包安装的文件清单
dpkg -S /etc/nginx/nginx.conf # 文件属于哪个包
dpkg --configure -a           # 修复未配置完的包

# rpm（RedHat 系）
rpm -ivh package.rpm          # 安装（v=verbose h=进度条）
rpm -qa | grep nginx          # 查询所有已装
rpm -ql nginx                 # 包文件清单
rpm -qf /etc/nginx/nginx.conf # 文件属于哪个包
rpm -e nginx                  # 卸载
rpm -V nginx                  # 校验完整性
```

## 仓库配置

```bash
# apt 源
cat /etc/apt/sources.list
ls /etc/apt/sources.list.d/
add-apt-repository ppa:nginx/stable    # 添加 PPA

# yum 源
cat /etc/yum.repos.d/nginx.repo
# [nginx-stable]
# name=nginx
# baseurl=https://nginx.org/packages/centos/$releasever/$basearch/
# gpgcheck=1
# enabled=1
```

## 源码编译

```bash
# 三步法：configure → make → make install
./configure --prefix=/usr/local/nginx --with-http_ssl_module
make -j$(nproc)               # -j 并行编译
make install

# 卸载只需删目录
rm -rf /usr/local/nginx

# 查看编译参数
nginx -V
```

| 方式 | 优点 | 缺点 |
|------|------|------|
| apt/yum | 依赖自动处理、方便升级 | 包版本可能旧 |
| dpkg/rpm | 离线可用、可控 | 依赖手动处理 |
| 源码编译 | 定制编译参数、版本最新 | 无自动升级、需手动管理 |

## 内核模块

```bash
lsmod                         # 已加载模块列表
modprobe ip_vs                # 加载模块
modprobe -r ip_vs             # 卸载模块
modinfo ip_vs                 # 模块详细信息
depmod                        # 重建模块依赖
```

## 实践场景

### 场景 1：离线环境装包

```bash
# 有网的机器下载依赖
apt-get download nginx && apt-cache depends -i nginx \
  | grep "Depends" | awk '{print $2}' | xargs apt-get download

# 或使用 apt-offline / yumdownloader
yumdownloader --resolve nginx
```

### 场景 2：内核升级

```bash
# Debian/Ubuntu
apt install linux-image-6.1.0-xxx

# CentOS/RHEL
yum install kernel-xxx
grub2-set-default 0 && grub2-mkconfig -o /boot/grub2/grub.cfg
reboot
```

### 场景 3：添加第三方仓库

```bash
# Docker
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | gpg --dearmor -o /usr/share/keyrings/docker.gpg
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | tee /etc/apt/sources.list.d/docker.list

# Nginx
wget https://nginx.org/keys/nginx_signing.key && apt-key add nginx_signing.key
echo "deb https://nginx.org/packages/ubuntu $(lsb_release -cs) nginx" >> /etc/apt/sources.list
```

### 场景 4：依赖修复

```bash
# dpkg 缺依赖
apt install -f                # 修复依赖
dpkg --configure -a           # 配置所有未配完的

# rpm 强制
rpm -e --nodeps pkg           # 跳过依赖卸载（慎用）
rpm -ivh --nodeps pkg.rpm     # 跳过依赖安装
```
