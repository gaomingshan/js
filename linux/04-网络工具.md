## curl —— 全能 HTTP 客户端

```bash
# 基本请求
curl http://example.com                      # GET 请求
curl -X POST http://api.example.com/login     # POST 请求

# 请求头与数据
curl -X POST http://api.example.com/login \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer token123" \
  -d '{"username":"admin","password":"pass"}'

# URL 编码 data（application/x-www-form-urlencoded）
curl -X POST http://api.example.com/login \
  -d "username=admin&password=pass"
```

### 调试利器

```bash
curl -v http://example.com                   # 看请求/响应全过程（请求头+响应头+响应体）
curl -i http://example.com                   # 只显示响应头 + 响应体
curl -w "\n%{http_code}\n" http://example.com # 只取 HTTP 状态码

# 更多 -w 变量
curl -w "HTTP: %{http_code}, Time: %{time_total}s, Size: %{size_download}bytes\n" \
  -o /dev/null -s http://example.com
```

### 常用选项

```bash
# 文件操作
curl -o output.html http://example.com       # 保存为指定文件名
curl -O http://example.com/file.tar.gz       # 保持原名下载
curl -C - -O http://example.com/bigfile.iso  # 断点续传

# 伪装请求
curl -A "Mozilla/5.0" http://example.com     # 自定义 User-Agent
curl -b "session=abc123" http://example.com  # 携带 Cookie
curl -e "https://google.com" http://example.com  # 伪装 Referer

# 安全与超时
curl -k https://self-signed.biz              # 忽略证书验证
curl --connect-timeout 5 http://example.com  # 连接超时 5 秒
curl --max-time 10 http://example.com        # 总超时 10 秒

# 跟随跳转
curl -L http://bit.ly/short-url              # 跟随 301/302 重定向

# 静默模式
curl -s http://example.com                   # 不显示进度和错误
curl -sS http://example.com                  # 只显示错误，不显示进度
```

### 上传文件

```bash
# multipart/form-data（模拟表单上传）
curl -F "file=@/path/to/photo.jpg" \
  -F "description=my photo" \
  http://api.example.com/upload

# PUT 上传
curl -T localfile.txt http://example.com/remote.txt
```

### 实战：API 接口测试

```bash
# 健康检查脚本
check_api() {
  code=$(curl -s -o /dev/null -w "%{http_code}" --connect-timeout 5 "$1")
  if [ "$code" = "200" ]; then
    echo "[OK] $1 → $code"
  else
    echo "[FAIL] $1 → $code"
    return 1
  fi
}

check_api "http://localhost:8080/health"
check_api "http://localhost:8081/health"
```

## wget —— 下载专用

```bash
wget http://example.com/file.tar.gz          # 基本下载
wget -c http://example.com/bigfile.iso       # 断点续传
wget --limit-rate=1m http://example.com/file # 限速 1MB/s
wget -b http://example.com/file              # 后台下载（写入 wget-log）
wget -O custom_name.tar.gz http://...        # 指定输出文件名

# 递归下载整个站点
wget -r -l 2 -np http://example.com/docs/    # 递归2层，不爬上级目录

# 镜像网站
wget -mk http://example.com                  # 适合离线浏览的镜像
```

### curl vs wget

| 场景 | curl | wget |
|------|------|------|
| HTTP POST/PUT/自定义头 | ✅ 首选 | ❌ 麻烦 |
| 文件下载 | ✅ | ✅ 首选（断点续传更强） |
| 递归下载整站 | ❌ | ✅ `-r` |
| 限速下载 | ✅ `--limit-rate` | ✅ `--limit-rate` |
| API 调试 | ✅ 首选 | ❌ |
| 多协议（FTP/SMTP/etc） | ✅ | ✅ |

## scp —— 简单传文件

```bash
# 上传
scp file.txt user@host:/home/user/          # 本地→远程
scp -r dir/ user@host:/home/user/           # 递归传目录

# 下载
scp user@host:/home/user/file.txt ./        # 远程→本地
scp -P 2222 file.txt user@host:             # 指定端口

# 限速
scp -l 1000 bigfile.tar.gz user@host:       # 限速 1000 Kb/s ≈ 125 KB/s
```

## rsync —— 增量同步

```bash
rsync -avz src/ user@host:/dest/            # 归档 + 压缩 + 显示详情
rsync -avz --delete src/ user@host:/dest/   # 同步且删除目标端多余文件
rsync -avz --exclude='*.log' src/ host:/dest/  # 排除文件
rsync -avz -e ssh src/ host:/dest/          # 指定 SSH（默认就是 SSH）

# 本地同步（速度快）
rsync -av src/ /backup/

# 试运行（--dry-run / -n）
rsync -avzn src/ user@host:/dest/           # 先看会同步什么，不真执行

# 增量备份 —— 多版本
rsync -avzb --backup-dir=/backup/prev/ src/ /backup/current/
```

### scp vs rsync

| 场景 | scp | rsync |
|------|-----|-------|
| 一次性传小文件 | ✅ 简单 | 也可 |
| 大文件/增量同步 | ❌ 每次都全量 | ✅ 只传差异 |
| 断点续传 | ❌ | ✅ |
| 自动删除多余文件 | ❌ | ✅ `--delete` |
| 排除模式 | ❌ | ✅ `--exclude` |

### 实战：部署新版本

```bash
# 打包 + 推送 + 解压 + 软链切换
tar -czf release.tar.gz dist/ package.json
scp release.tar.gz deploy@host:/opt/myapp/
ssh deploy@host "cd /opt/myapp && \
  tar -xzf release.tar.gz -C releases/$(date +%Y%m%d_%H%M%S) && \
  ln -snf releases/$(date +%Y%m%d_%H%M%S) current"

# rsync 版本（更高效，只增量同步变更文件）
rsync -avz --delete dist/ deploy@host:/opt/myapp/current/
```

## 端口探测

```bash
# telnet —— 最直接，能连就说明端口开了
telnet 192.168.1.1 80
# 输出示例：
# Trying 192.168.1.1...
# Connected to 192.168.1.1.

# nc（netcat）—— 更灵活
nc -zv host 80                             # -z 只扫描不连，-v 详情
nc -zv host 22 80 443                      # 多端口
nc -zv host 1-1024                         # 扫描范围

# nmap —— 全端口扫描
nmap -p 80,443 host                        # 指定端口
nmap -p 1-1000 host                        # 范围
nmap -p- host                              # 全端口 1-65535（慢）
nmap -sV host                              # 探测服务版本
nmap -O host                               # 探测操作系统

# ss —— 本机监听端口（替代 netstat）
ss -tlnp                                   # TCP 监听 + 进程名
ss -ulnp                                   # UDP 监听
ss -tlnp | grep :80                        # 看谁占用了 80 端口

# 旧工具 netstat（逐渐被 ss 取代）
netstat -tlnp
```

### 端口探测对比

| 工具 | 场景 | 优点 | 缺点 |
|------|------|------|------|
| `telnet` | 快速确认端口 | 无需安装，简单 | 连上就退不了（Ctrl+] quit） |
| `nc -zv` | 批量端口检查 | 可写脚本，`-w` 超时 | 大量端口较慢 |
| `nmap` | 全端口扫描 | 功能最强 | 大范围扫描慢，可能被 IDS 发现 |
| `ss` | 查本机端口 | 快速，显示进程 | 不能扫远程 |

## 域名解析

```bash
# dig —— DNS 查询首选
dig example.com                              # 完整输出
dig example.com +short                       # 简洁结果
dig example.com A                            # A 记录
dig example.com MX                           # MX 记录
dig example.com NS                           # NS 记录
dig @8.8.8.8 example.com                     # 指定 DNS 服务器
dig -x 8.8.8.8                               # 反向解析（IP→域名）

# nslookup —— 交互式
nslookup example.com
nslookup example.com 8.8.8.8                 # 指定 DNS

# host —— 最简
host example.com
host 8.8.8.8                                # 反向解析

# getent —— 查本地 /etc/hosts（不走 DNS）
getent hosts my-server                       # 查 hosts 文件
getent ahosts my-server                      # 查所有地址
```

## 实践场景

### 场景 1：API 健康检查 + 告警

```bash
#!/bin/bash
# 每 30 秒检测一次，连续 3 次失败则重启服务
fail=0
while true; do
  code=$(curl -s -o /dev/null -w "%{http_code}" --connect-timeout 5 http://localhost:8080/health)
  if [ "$code" != "200" ]; then
    ((fail++))
    echo "Fail #$fail"
    if [ "$fail" -ge 3 ]; then
      echo "Restarting service..."
      systemctl restart myapp
      fail=0
    fi
  else
    fail=0
  fi
  sleep 30
done
```

### 场景 2：跨服务器日志同步

```bash
# 生产→分析服务器，每 5 分钟增量同步
rsync -avz --remove-source-files \
  /var/log/myapp/ \
  analyst@log-server:/data/logs/ \
  --include="*.log" --exclude="*.gz"

# 或者用 rsync daemon 模式（走 rsync 协议更高效）
rsync -avz /var/log/ rsync://log-server/logs/
```

### 场景 3：全端口扫描找服务

```bash
# 扫描内网服务器开放的端口，输出到文件
nmap -p- -T4 192.168.1.100 -oG open_ports.txt

# 提取开放端口
grep -oP '\d+/open' open_ports.txt | cut -d/ -f1 | paste -sd,

# 快速查看 MySQL 是否可达
nc -zv db-server 3306 && echo "MySQL ready" || echo "MySQL down"
```

### 场景 4：curl 模拟浏览器请求

```bash
# 完整模拟登录流程
# 1. GET 登录页获取 Cookie
curl -c cookies.txt -s https://example.com/login > /dev/null

# 2. POST 登录
curl -b cookies.txt -c cookies.txt \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "user=admin&password=secret" \
  -L https://example.com/login

# 3. 访问需鉴权的页面
curl -b cookies.txt https://example.com/dashboard
```

### 场景 5：快速排障 —— 链路检查四步

```bash
# 第1步：DNS 解析是否正常？
dig +short myapp.internal

# 第2步：端口是否可达？
nc -zv myapp.internal 443

# 第3步：HTTP 响应是否正常？
curl -I https://myapp.internal/health

# 第4步：耗时如何？
curl -w "dns: %{time_namelookup}s, connect: %{time_connect}s, ssl: %{time_appconnect}s, total: %{time_total}s\n" \
  -o /dev/null -s https://myapp.internal/health
```
