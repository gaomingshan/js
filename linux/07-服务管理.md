> systemd/journalctl/进程管理 | 守护进程的全生命周期管理

## systemd unit 文件

### 服务单元模板

路径：`/etc/systemd/system/myapp.service`

```ini
[Unit]
Description=My App Service
Documentation=https://wiki.example.com
After=network.target postgresql.service
Requires=postgresql.service         # 强依赖，postgresql 必须启动
Wants=redis.service                 # 弱依赖，redis 启动失败不影响

[Service]
Type=simple                         # simple/forking/oneshot/notify/dbus
User=appuser
Group=appuser
WorkingDirectory=/opt/app
EnvironmentFile=-/etc/myapp/env     # 环境变量文件（- 表示文件存在则加载）
Environment=JAVA_HOME=/usr/lib/jvm/java-17
ExecStart=/usr/bin/java -jar app.jar
ExecStop=/usr/bin/java -jar app.jar --stop
ExecReload=/bin/kill -HUP $MAINPID
Restart=on-failure                  # always/on-failure/no/on-abnormal
RestartSec=5
StartLimitIntervalSec=60
StartLimitBurst=3                   # 60 秒内最多重启 3 次
LimitNOFILE=65536
LimitNPROC=4096
TimeoutStartSec=60
TimeoutStopSec=30

[Install]
WantedBy=multi-user.target
```

### Type 说明

| Type | 适用场景 |
|------|---------|
| `simple` | 前台进程，主进程持续运行（默认） |
| `forking` | 传统 daemon（启动后 fork 退出的子进程，需 PIDFile） |
| `oneshot` | 一次性任务（如启动前清理） |
| `notify` | 进程通过 sd_notify 通知 systemd 已就绪 |
| `dbus` | 等待 D-Bus 总线就绪 |

### Timer 单元（替代 cron）

```ini
# /etc/systemd/system/cleanup.timer
[Unit]
Description=Weekly cleanup timer

[Timer]
OnCalendar=weekly
Persistent=true                    # 错过时间后立即执行
RandomizedDelaySec=600             # 随机延迟防雪崩

[Install]
WantedBy=timers.target
```

```ini
# /etc/systemd/system/cleanup.service
[Unit]
Description=Weekly cleanup job

[Service]
Type=oneshot
ExecStart=/usr/local/bin/cleanup.sh
```

```bash
systemctl enable --now cleanup.timer
systemctl list-timers
```

## systemctl

```bash
# 基础管理
systemctl start myapp
systemctl stop myapp
systemctl restart myapp
systemctl reload myapp           # 发 SIGHUP 重载配置
systemctl status myapp           # 状态 + 最近日志
systemctl enable myapp           # 开机自启
systemctl enable --now myapp     # 启用并立即启动
systemctl disable myapp
systemctl is-active myapp        # active/inactive
systemctl is-enabled myapp       # enabled/disabled

# 重载配置
systemctl daemon-reload          # 改 unit 文件后必须执行

# 查看
systemctl list-units --type=service --state=running
systemctl list-units --type=service --state=failed
systemctl cat myapp              # 查看当前 unit 内容
systemctl show myapp             # 查看所有属性
systemctl list-dependencies myapp  # 查看依赖树

# 屏蔽/掩藏
systemctl mask myapp             # 完全禁用（link 到 /dev/null）
systemctl unmask myapp
```

### systemctl 与 service 对比

| 操作 | 旧 sysvinit | systemd |
|------|------------|---------|
| 启动 | `service nginx start` | `systemctl start nginx` |
| 开机自启 | `chkconfig nginx on` | `systemctl enable nginx` |
| 查看状态 | `service nginx status` | `systemctl status nginx` |
| 查看全部 | `chkconfig --list` | `systemctl list-unit-files` |

## journalctl

```bash
# 按服务
journalctl -u myapp                    # 查看所有日志
journalctl -u myapp -f                 # 实时跟踪（类似 tail -f）
journalctl -u myapp --since "1 hour ago"
journalctl -u myapp --since "2026-01-01" --until "2026-01-02"

# 输出格式
journalctl -u myapp -n 50              # 最近 50 行
journalctl -u myapp -x                 # 附带错误说明
journalctl -u myapp -o json-pretty     # JSON 格式
journalctl -u myapp -o short-iso       # 含 ISO 时间戳

# 按优先级
journalctl -u myapp -p err             # emerg/alert/crit/err/warning/notice/info/debug
journalctl -u myapp -p err -g "OOM"    # 按优先级加 grep

# 磁盘管理
journalctl --disk-usage                # 查看占用空间
journalctl --vacuum-size=200M          # 限制总大小
journalctl --vacuum-time=30d           # 保留 30 天

# 持久化（默认 /var/log/journal）
# 修改 /etc/systemd/journald.conf: Storage=persistent
# systemctl restart systemd-journald
```

### journalctl 配置

```ini
# /etc/systemd/journald.conf
[Journal]
Storage=persistent                    # auto/volatile/persistent/none
Compress=yes
Seal=yes
SystemMaxUse=500M
SystemMaxFileSize=100M
MaxRetentionSec=30day
ForwardToSyslog=no
```

## 进程管理

### ps

```bash
ps aux                                # 所有进程（BSD 格式）
ps -ef                                # 所有进程（标准格式）
ps -ef --forest                       # 树状父子关系
ps -eo pid,ppid,cmd,%mem,%cpu --sort=-%mem | head  # 自定义列+排序

# 找特定进程
ps aux | grep java
pgrep -l java                         # 只找 PID + 名字
pgrep -f "java -jar app.jar"          # 按完整命令行
pidof nginx                           # 找所有 PID
```

### top / htop

```bash
top                                   # 默认按 CPU 排序
top -o %MEM                           # 按内存排序
top -p $(pgrep -d',' java)            # 只监视特定进程
top -bn1 | head -20                   # 一次快照（批量用）

htop                                  # 增强版（建议安装）
```

### kill

```bash
kill -TERM 1234                       # SIGTERM，优雅终止（默认）
kill -INT 1234                        # SIGINT，类似 Ctrl+C
kill -HUP $(cat /var/run/nginx.pid)   # SIGHUP，重载配置
kill -USR1 PID                        # SIGUSR1，程序自定义
kill -9 1234                          # SIGKILL，强制杀死（最后手段）
kill -0 1234                          # 检查进程是否存在（不发信号）

pkill -f "java -jar app.jar"          # 按命令行匹配杀死
pkill -TERM -u appuser                # 杀死某用户所有进程
killall nginx                         # 按名字杀死全部
```

### 信号表

| 信号 | 编号 | 用途 |
|------|------|------|
| HUP  | 1    | 重载配置 |
| INT  | 2    | 中断（Ctrl+C） |
| QUIT | 3    | 退出并生成 core dump |
| KILL | 9    | 强制杀死（进程无法捕获） |
| TERM | 15   | 优雅终止（默认） |
| USR1 | 10   | 程序自定义 |
| USR2 | 12   | 程序自定义 |

## 后台运行

```bash
# nohup — 忽略 HUP 信号
nohup java -jar app.jar > app.log 2>&1 &
disown %1                             # 从当前 shell 任务表移除

# screen — 虚拟终端
screen -S mysession                   # 创建
screen -ls                            # 列出
screen -r mysession                   # 恢复
# Ctrl+A D                             # 脱离
# Ctrl+A [                             # 进入滚动模式

# tmux — 功能更强的 screen
tmux new -s mysession
tmux ls
tmux attach -t mysession
# Ctrl+B D                             # 脱离
# Ctrl+B [                             # 滚动模式
# Ctrl+B "                             # 水平分割
# Ctrl+B %                             # 垂直分割
```

### nohup vs disown vs screen/tmux

| 方式 | 终端退出后存活 | 重连 | 多窗口 |
|------|--------------|------|-------|
| `nohup` + `&` | 是 | 否 | 否 |
| `disown` | 是 | 否 | 否 |
| `screen` | 是 | 是 | 是 |
| `tmux` | 是 | 是 | 是（更强大） |

## 实践场景

### 场景 1：Java 应用 systemd unit

```ini
# /etc/systemd/system/myapp.service
[Unit]
Description=My Java Application
After=network.target postgresql.service
Requires=postgresql.service

[Service]
Type=simple
User=appuser
WorkingDirectory=/opt/myapp
ExecStart=/usr/bin/java -Xms512m -Xmx2g \
  -jar /opt/myapp/app.jar \
  --spring.profiles.active=prod
ExecStop=/bin/kill -TERM $MAINPID
Restart=on-failure
RestartSec=10
StartLimitIntervalSec=60
StartLimitBurst=3
LimitNOFILE=65536
TimeoutStopSec=30

[Install]
WantedBy=multi-user.target
```

```bash
# 部署步骤
systemctl daemon-reload
systemctl enable --now myapp
systemctl status myapp

# 查看资源占用
systemctl show myapp -p MemoryCurrent,CPUUsageNSec
```

### 场景 2：logrotate + journalctl 日志管理

```bash
# 应用日志轮转
# /etc/logrotate.d/myapp
/var/log/myapp/*.log {
    daily
    rotate 30
    compress
    delaycompress
    missingok
    notifempty
    copytruncate                     # 拷贝后截断（不重启进程）
    postrotate
        systemctl reload myapp 2>/dev/null || true
    endscript
}

# journalctl 磁盘限制
# /etc/systemd/journald.conf
SystemMaxUse=500M
SystemMaxFileSize=100M
MaxRetentionSec=30day

# 配合 cron 清理
# /etc/cron.daily/log-cleanup
#!/bin/bash
journalctl --vacuum-size=500M 2>/dev/null
find /var/log -name "*.log.*.gz" -mtime +30 -delete
```

### 场景 3：进程监控 + 自动拉起

```bash
#!/bin/bash
# /usr/local/bin/process-watchdog.sh
# 检查进程是否存在，不存在则重启

APP_NAME="myapp"
SERVICE_NAME="myapp.service"
CHECK_INTERVAL=30
LOG_FILE="/var/log/watchdog.log"

log() {
  echo "$(date '+%Y-%m-%d %H:%M:%S') $1" >> "$LOG_FILE"
}

while true; do
  if ! systemctl is-active --quiet "$SERVICE_NAME"; then
    log "WARN: $APP_NAME 已停止，尝试重启"
    systemctl start "$SERVICE_NAME"
    sleep 5

    # 二次确认
    if systemctl is-active --quiet "$SERVICE_NAME"; then
      log "OK: $APP_NAME 已恢复"
    else
      log "ERROR: $APP_NAME 启动失败，尝试杀死残留进程"
      pkill -f "$APP_NAME" 2>/dev/null
      systemctl start "$SERVICE_NAME"

      # 仍失败则触发告警
      if ! systemctl is-active --quiet "$SERVICE_NAME"; then
        log "CRIT: $APP_NAME 仍然无法启动，发送告警"
        echo "服务 $APP_NAME 多次启动失败" | mail -s "[CRIT] 服务异常" ops@example.com
      fi
    fi
  fi
  sleep "$CHECK_INTERVAL"
done
```

```bash
# 配合 systemd timer 运行
# /etc/systemd/system/watchdog.service
[Unit]
Description=Process Watchdog

[Service]
Type=simple
ExecStart=/usr/local/bin/process-watchdog.sh
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```
