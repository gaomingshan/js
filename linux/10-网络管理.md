## IP 配置

### ip —— 新一代网络命令（推荐）

```bash
ip addr show eth0             # 查看 IP
ip addr add 10.0.0.2/24 dev eth0     # 添加 IP
ip addr del 10.0.0.2/24 dev eth0     # 删除 IP
ip link set eth0 up           # 启用网卡
ip link set eth0 down         # 禁用网卡
ip link set eth0 mtu 9000     # 改 MTU
```

### 新旧命令对比

| 新命令（iproute2） | 旧命令（net-tools） |
|----|----|
| `ip addr` | `ifconfig` |
| `ip route` | `route -n` |
| `ip neigh` | `arp -a` |
| `ip link` | `ifconfig` / `mii-tool` |
| `ss -tlnp` | `netstat -tlnp` |
| `ip netns` | — |

> `net-tools` 已停止维护，**新系统应使用 `iproute2`**。

### 网络命名空间

```bash
ip netns add ns1              # 创建
ip netns exec ns1 bash        # 进入
ip link set veth0 netns ns1   # 移入
```

## 路由

### 路由查看

```bash
ip route show                 # 查看路由表
ip route show default         # 只看默认路由
ip route get 8.8.8.8          # 测试到某 IP 走哪条路由

# 旧命令
route -n                      # 查看路由表（数字格式）
```

### 路由操作

```bash
# 添加默认网关
ip route add default via 10.0.0.1
ip route add default via 10.0.0.1 dev eth0

# 添加静态路由
ip route add 10.0.0.0/8 via 10.0.0.1 dev eth0
ip route add 172.16.0.0/12 via 10.0.0.1

# 删除路由
ip route del 10.0.0.0/8 via 10.0.0.1

# 永久配置写入 /etc/network/interfaces 或 /etc/sysconfig/network-scripts/
```

### 策略路由

```bash
# 两个网口走不同网关
echo 200 table2 >> /etc/iproute2/rt_tables
ip rule add from 10.0.0.0/24 table table2
ip route add default via 10.0.0.1 dev eth0 table table2
```

## DNS

### resolv.conf

```bash
# /etc/resolv.conf
nameserver 8.8.8.8
nameserver 114.114.114.114
search example.com          # 补全域名后缀
options timeout:2 attempts:1 rotate
```

### systemd-resolved

```bash
resolvectl status             # 当前 DNS 状态
resolvectl dns eth0 8.8.8.8   # 设置网卡 DNS
resolvectl domain eth0 example.com
resolvectl query google.com   # 解析测试

# 启用/禁用
systemctl enable --now systemd-resolved
# /etc/resolv.conf → 软链到 /run/systemd/resolve/stub-resolv.conf
```

### 域名解析排查

```bash
nslookup google.com           # 传统
dig google.com                # 详细 DNS 响应
dig +short google.com         # 只看 IP

host google.com               # 简洁
getent hosts google.com       # 走系统 nsswitch（含 hosts 文件）

# 反向解析
dig -x 8.8.8.8
nslookup 8.8.8.8
```

### /etc/hosts

```
127.0.0.1  localhost
10.0.0.5   db-master.myapp.internal
```

优先于 DNS（取决于 `/etc/nsswitch.conf` 中 `hosts: files dns` 顺序）。

## 防火墙

### nftables（新）

```bash
nft list ruleset                          # 查看全部规则
nft add rule inet filter input tcp dport 443 accept
nft add rule inet filter input ip saddr 10.0.0.0/8 accept
nft add rule inet filter input drop        # 默认拒绝

nft flush ruleset                          # 清空所有规则

# 持久化
nft list ruleset > /etc/nftables.conf
```

### iptables（旧）

```bash
iptables -L -n -v                   # 查看规则（verbose）
iptables -A INPUT -p tcp --dport 22 -j ACCEPT        # 开放 22
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT  # 已有连接
iptables -A INPUT -j DROP                             # 默认拒绝

iptables -D INPUT 3                  # 删除第 3 条规则
iptables -F                          # 清空
iptables -P INPUT ACCEPT             # 改默认策略

# NAT 端口转发（PREROUTING）
iptables -t nat -A PREROUTING -p tcp --dport 80 -j REDIRECT --to-port 8080
iptables -t nat -A PREROUTING -p tcp --dport 80 -j DNAT --to-destination 10.0.0.2:8080
```

### firewalld（CentOS/RHEL）

```bash
# 简单操作
firewall-cmd --add-port=80/tcp --permanent
firewall-cmd --add-service=http --permanent
firewall-cmd --reload

# 详细
firewall-cmd --list-all
firewall-cmd --zone=public --add-source=10.0.0.0/8
```

### ufw（Ubuntu）

```bash
ufw allow 22/tcp
ufw allow 443
ufw allow from 10.0.0.0/8 to any port 3306
ufw enable
ufw status verbose
```

### 防火墙对比

| 工具 | 适用 | 特点 |
|------|------|------|
| `nftables` | 新系统（>= RHEL8/Debian10） | 统一 ipv4/ipv6，性能好 |
| `iptables` | 旧系统 | 经典，模块多 |
| `firewalld` | CentOS/RHEL | 动态 zone，D-Bus |
| `ufw` | Ubuntu/Debian | 简单易用 |

## 抓包

### tcpdump

```bash
tcpdump -i eth0 port 80 -nn           # 抓 HTTP 流量
tcpdump -i eth0 host 10.0.0.1 -X      # 抓某 IP 报文 + hex
tcpdump -i eth0 -w capture.pcap       # 保存到文件（后续用 Wireshark 看）
tcpdump -r capture.pcap               # 读文件
tcpdump -c 100                        # 抓 100 个包后退出
tcpdump -s 0                          # 抓完整包（默认 262144 字节）

# 过滤技巧
tcpdump 'tcp[tcpflags] & tcp-syn != 0'   # 只抓 SYN
tcpdump 'tcp port 80 and (tcp[((tcp[12] & 0xf0) >> 2)] = 0x47)'  # 抓 GET
tcpdump -i any 'icmp'                    # 抓 ping

# 跳过域名解析（更快、更安全）
tcpdump -nn -v
```

### tcpdump 表达式

| 表达式 | 含义 |
|--------|------|
| `port 80` | 端口 80 |
| `host 1.2.3.4` | 某 IP |
| `net 10.0.0.0/8` | 某网段 |
| `tcp` / `udp` / `icmp` | 协议 |
| `src` / `dst` | 源/目标 |
| `and` / `or` / `not` | 逻辑组合 |
| `tcp[tcpflags] & tcp-syn != 0` | SYN 包 |

### 常用抓包场景

```bash
# 看 TCP 握手
tcpdump -i eth0 host 10.0.0.1 and port 443 -nn

# 看 DNS 查询
tcpdump -i eth0 port 53 -nn

# 看 DHCP
tcpdump -i eth0 port 67 or port 68 -nn

# 看重传
tcpdump -i eth0 'tcp[tcpflags] & (tcp-retrans) != 0'
```

## 网卡性能

### ethtool —— 网卡硬件信息

```bash
ethtool eth0               # 速率/双工/自协商
ethtool -i eth0            # 驱动版本
ethtool -S eth0            # 网卡统计（丢包/错包/rx_crc_error）
ethtool -k eth0            # offload 功能（gro/gso/tso）
ethtool -g eth0            # ring buffer 大小
ethtool --set-ring eth0 rx 4096 tx 4096   # 改 ring buffer
```

### ip -s —— 内核统计

```bash
ip -s link show eth0       # 内核收发统计
ip -s -s link show eth0    # 更多详情

# 重点关注
# RX errors / dropped / overrun / frame
# TX errors / dropped / carrier / collisions
```

### ss —— 套接字统计

```bash
ss -tlnp                   # TCP 监听端口（替代 netstat）
ss -tlnp | column -t       # 格式化
ss -ti                     # TCP 连接详情（cwnd/rtt/mss）
ss -s                      # 汇总统计
ss -t -o state established '( dport = :http or sport = :http )'  # 高级过滤
```

`socket` 状态：

| 状态 | 含义 |
|------|------|
| `LISTEN` | 监听 |
| `ESTAB` | 已连接 |
| `TIME-WAIT` | 等待关闭 |
| `CLOSE-WAIT` | 对端关闭，本端未关 |
| `FIN-WAIT-1/2` | 主动关闭 |
| `SYN-SENT` / `SYN-RECV` | 握手 |

### sar —— 网卡流量

```bash
sar -n DEV 1              # 每秒网卡流量
sar -n TCP,SOCK 1         # TCP 连接 + 套接字
sar -n EDEV 1             # 错误统计

# 历史
sar -n DEV -f /var/log/sa/sa01
```

## 网卡绑定（bonding）

### bond 模式

| 模式 | 值 | 说明 |
|------|----|------|
| balance-rr | 0 | 轮询（高可用+负载均衡） |
| active-backup | 1 | 主备（最常用） |
| balance-xor | 2 | 基于 XOR 的负载 |
| 802.3ad | 4 | LACP（交换机需支持） |
| balance-tlb | 5 | 发送端负载均衡 |
| balance-alb | 6 | 收发负载均衡 |

### 配置示例

```bash
# Debian/Ubuntu /etc/network/interfaces
auto bond0
iface bond0 inet static
    address 10.0.0.100/24
    gateway 10.0.0.1
    bond-slaves eth0 eth1
    bond-mode 802.3ad
    bond-miimon 100
    bond-lacp-rate fast

# RHEL/CentOS /etc/sysconfig/network-scripts/
# ifcfg-bond0 + ifcfg-eth0/eth1
```

### 查看绑定状态

```bash
cat /proc/net/bonding/bond0
# - Mode: 802.3ad
# - MII Status: up
# - Active Slave: eth0
# - Slave Interface: eth0 / eth1 状态
```

### 网卡聚合对比（bond vs team）

| 特性 | bonding | teaming |
|------|---------|---------|
| 内核版本 | 2.0+ | 3.3+ |
| 配置方式 | `/proc` / sysfs | `teamd` 守护进程 |
| 灵活性 | 一般 | 高（JSON 配置） |
| 兼容性 | 广泛 | RHEL7+ |

## 实战场景

### 场景 1：端口不通全链路排查

```bash
# 1. 本机是否监听
ss -tlnp | grep 3306

# 2. 防火墙是否放行
iptables -L -n | grep 3306
nft list ruleset | grep 3306

# 3. 本机是否可以访问自己
curl -v telnet://localhost:3306

# 4. 本机到远端是否可达
ping 10.0.0.5
traceroute 10.0.0.5
telnet 10.0.0.5 3306

# 5. 抓包看 TCP 握手是否完成
tcpdump -i eth0 host 10.0.0.5 and port 3306 -nn
# 如果只看到 SYN → 无 SYN-ACK → 远程没在监听或被防火墙拦
# 如果 SYN → SYN-ACK → RST → 端口通了但应用拒绝

# 6. 远程检查
ssh remote "ss -tlnp | grep 3306"
ssh remote "iptables -L -n | grep 3306"
```

### 场景 2：iptables 端口转发到内网

```bash
# 外网访问 2222 → 内网机器 192.168.1.100:22
iptables -t nat -A PREROUTING -p tcp --dport 2222 \
  -j DNAT --to-destination 192.168.1.100:22

# 转发回包（必要）
iptables -t nat -A POSTROUTING -d 192.168.1.100 -p tcp --dport 22 \
  -j SNAT --to-source 10.0.0.1

# 开启 IP 转发
echo 1 > /proc/sys/net/ipv4/ip_forward

# 持久化
iptables-save > /etc/iptables/rules.v4
```

### 场景 3：网卡丢包问题排查

```bash
# 1. 看丢包
ip -s link show eth0        # dropped 是否增长
ethtool -S eth0 | grep -E 'drop|error|miss'

# 2. 常见原因
# a) ring buffer 满
ethtool -g eth0
ethtool --set-ring eth0 rx 4096 tx 4096

# b) 中断不均（多队列）
cat /proc/interrupts | grep eth0

# c) 软中断耗时
cat /proc/softirqs | grep NET_RX
# 设置 RPS（Receive Packet Steering）
echo f > /sys/class/net/eth0/queues/rx-0/rps_cpus

# d) 驱动 offload 问题
ethtool -k eth0
ethtool -K eth0 gro off gso off tso off  # 关掉后对比

# e) 防火墙规则过多
time nft list ruleset   # 看耗时
```

### 场景 4：查看当前网络瓶颈

```bash
# 1. 带宽
sar -n DEV 1 5
# rxkB/s + txkB/s → 接近网卡速率说明带宽打满

# 2. TCP 重传
netstat -s | grep -i retrans
ss -ti | grep -c retrans
# 重传率高 → 网络拥塞/不稳定

# 3. 连接数
ss -s
ss -tan | awk '{print $1}' | sort | uniq -c

# 4. TIME_WAIT 太多
ss -tan state time-wait | wc -l
sysctl -w net.ipv4.tcp_tw_reuse=1      # 复用
sysctl -w net.ipv4.tcp_fin_timeout=15  # 减少等待

# 5. 连接队列溢出
cat /proc/net/netstat | grep ListenOverflows
ss -lnt | column -t   # Send-Q 表示 backlog
```

### 场景 5：TCP 连接状态汇总

```bash
# 各状态数量
ss -tan | awk '{print $1}' | sort | uniq -c

# CLOSE_WAIT 排查（应用没 close）
ss -tan state close-wait
# 确认应用代码是否忘了 close / 连接池泄漏

# SYN_RECV 过多（半连接队列满）
ss -tan state syn-recv
sysctl -w net.ipv4.tcp_max_syn_backlog=8192
sysctl -w net.ipv4.tcp_syncookies=1
```
