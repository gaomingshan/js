# 文件与文本操作

> 日常操作 | 查询/浏览/查找/链接 | 最基础也最常用

---

## 文件浏览

### ls —— 最常用的查看命令

```bash
ls -l                 # 详细格式（权限/硬链数/属主/大小/时间/名字）
ls -la                # 包含隐藏文件（以 . 开头）
ls -lh                # 大小以 K/M/G 显示
ls -lt                # 按修改时间排序，最新在前
ls -ltr               # 按修改时间排序，最旧在前
ls -lS                # 按文件大小排序
ls -R                 # 递归子目录
```

### tree —— 目录树结构

```bash
tree -d               # 只看目录
tree -L 2             # 只显示两层深度
tree -h               # 显示文件大小
tree -I node_modules  # 排除文件夹
```

### stat —— 文件元数据

```bash
stat file.txt
# 输出：大小、块数、Inode、权限、链接数、Access/Modify/Change 时间
```

### file —— 识别文件类型

```bash
file somefile          # 不管扩展名，读魔数判断类型
file -i somefile       # MIME 类型（如 text/plain, image/png）
```

## 文件查看

### cat —— 拼接输出

```bash
cat file               # 看小文件
cat -n file            # 显示行号
cat -A file            # 显示不可见字符（制表符=^I、换行=$）
cat file1 file2 > out  # 合并文件
```

### more / less —— 分页查看

less 比 more 功能多，**日常用 less**：

```bash
less file              # 翻页查看
# 快捷键：
#   Space/PageDown   下翻一屏
#   b/PageUp         上翻一屏
#   j/k              上下翻一行
#   g                跳到第一行
#   G                跳到最后一行
#   /pattern         向下搜索
#   ?pattern         向上搜索
#   n/N              下一个/上一个匹配
#   q                退出
```

### head / tail —— 头尾

```bash
head -20 file          # 前 20 行
tail -20 file          # 后 20 行
tail -f file           # 实时跟踪追加（看日志最常用）
tail -F file           # 比 -f 强：文件被截断/轮转后自动重跟（生产推荐）
tail -n +100 file      # 从第 100 行开始到结尾
```

### tac / rev —— 反向

```bash
tac file               # 行顺序反转（cat 倒过来）
rev file               # 每行字符反转
```

## 文件查找

### find —— 最强查找

```bash
find . -name "*.log"                     # 按名字
find . -type f -size +100M               # 大于 100MB 的文件
find . -type f -mtime -3                 # 3 天内修改过
find . -type d -empty                    # 空目录
find . -perm 644                         # 精确权限
find . -user www-data                    # 属于某个用户

# 找到后执行操作
find . -name "*.tmp" -delete             # 删
find . -name "*.log" -exec gzip {} \;    # 逐个压缩
find . -name "*.log" -mtime +7 -delete   # 删 7 天前日志

# 配合 xargs（批量执行，性能更好）
find . -type f -name "*.log" -mtime +7 | xargs rm -f
find . -type f -name "*.java" | xargs grep "TODO"
```

### locate —— 快速查找（依赖数据库）

```bash
locate nginx.conf       # 秒出结果
locate -i error_log     # 忽略大小写
sudo updatedb           # 更新数据库
```

### which / whereis —— 找命令路径

```bash
which nginx             # 找到 PATH 中的位置
whereis nginx           # 找二进制、源码、man（路径更广）
```

## 文件操作

```bash
cp src dst              # 复制
cp -r src/ dst/         # 递归复制目录
cp -p src dst           # 保持属性（权限/时间/属主）
cp -a src dst           # 归档模式（-dR --preserve=all）

mv old new              # 重命名/移动

rm file                 # 删文件
rm -r dir/              # 删目录树
rm -f file              # 强制删（不询问）
# ⚠️ rm -rf /          # 绝对不要执行

mkdir dir               # 建目录
mkdir -p a/b/c          # 递归创建

touch file              # 创建空文件/更新时间戳
touch -t 202501010000 file  # 设置指定时间

install -d /path        # mkdir -p 的另一种写法
install -m 755 src /dst # 复制并设置权限
```

### 批量改名

```bash
# 后缀 .txt → .md
rename 's/\.txt$/\.md/' *.txt

# 所有文件改为小写
rename 'y/A-Z/a-z/' *

# 加前缀
for f in *.log; do mv "$f" "backup_$f"; done
```

### 按时间删除旧文件

```bash
# 删 7 天前的日志
find /var/log/myapp -name "*.log" -mtime +7 -delete

# 删 30 天前的备份
find /backup -name "*.tar.gz" -mtime +30 -exec rm {} \;
```

## 链接管理

Linux 有两种链接：**硬链接**和**符号链接（软链接）**。

```
硬链接：ln a b
  两个文件名指向同一个 Inode
  a 和 b 地位平等，删一个不影响另一个
  不能跨文件系统、不能链目录

符号链接：ln -s a b
  b 指向 a 的文件路径
  删 a → b 变成"断链"
  可以跨文件系统、可以链目录
```

```bash
ln -s /usr/local/nginx/sbin/nginx /usr/local/bin/nginx   # 做软链（最常用）
ln a.txt b.txt          # 硬链接


# 查看
ls -li                  # 第一列是 Inode 号，硬链接共享
readlink link_name      # 查看软链接指向的目标

# 场景
# 应用版本切换：ln -snf app-v2.0 app-current
# 日志路径映射：ln -s /data/logs /var/log/myapp
```

## 内容比较

```bash
diff file1 file2                    # 逐行比较
diff -u file1 file2                 # uniform 格式（更可读，patch 用）
diff -r dir1/ dir2/                 # 递归比较目录

sdiff file1 file2                   # 并排显示

comm file1 file2                    # 三列：file1独有、file2独有、共有
comm -12 file1 file2                # 只看共有行（即交集）

cmp -l file1 file2                  # 按字节比较，显示不同位置
```

### patch 打补丁

```bash
diff -u old new > fix.patch        # 生成补丁文件
patch -p1 < fix.patch               # 应用补丁
patch -RE < fix.patch               # 撤回补丁
```

## 实践场景

### 场景 1：排查大文件

```bash
# 查看磁盘占用最大的目录
du -sh /* 2>/dev/null | sort -rh | head -10

# 查看当前目录最大的 20 个文件
find . -type f -exec du -Sh {} + | sort -rh | head -20

# 再具体点：只查 /var/log 下大于 500M 的文件
find /var/log -type f -size +500M -exec ls -lh {} \;
```

### 场景 2：日志实时跟踪 + 过滤

```bash
# 实时看错误
tail -f app.log | grep "ERROR"

# 从 1 小时前开始看
tail -f -n +$(awk 'BEGIN {c=0} /2026-01-01 12:00:00/{c=NR} END {print c}' app.log) app.log

# 更简单的：journalctl 按时间
journalctl -u myapp --since "1 hour ago" -f
```

### 场景 3：批量替换配置

```bash
# 找到所有 .conf 文件，将 8080 替换为 9090
find /etc -name "*.conf" -exec sed -i 's/:8080/:9090/g' {} \;

# 或者用 grep 确认一下再替换
grep -rl "8080" /etc/ --include="*.conf" | xargs sed -i 's/:8080/:9090/g'
```

### 场景 4：两个目录同步

```bash
# src 和 dst 对比，dst 多出来的删掉，缺的补上
diff -rq src/ dst/ | grep "Only in src" | awk '{print $4}' | xargs -I{} cp {} dst/
```

### 场景 5：批量压缩归档旧日志

```bash
# 将 30 天前的 .log 逐个压缩
find /var/log -name "*.log" -mtime +30 -exec gzip {} \;

# 打包成一个 tar
find /var/log -name "*.log" -mtime +30 -print0 | xargs -0 tar -czf old-logs.tar.gz
```

新服务器初始化的 `ls` 别名建议加到 `.bashrc`：

```bash
alias ll='ls -lh'
alias la='ls -lha'
alias lt='ls -ltr'
alias l='ls -CF'
```
