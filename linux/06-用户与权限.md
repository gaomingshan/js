# 用户与权限

> 用户/组/sudo/文件权限/ACL | 多用户系统的安全基础

---

## 用户管理

### 创建/修改/删除

```bash
# 创建
useradd -m -s /bin/bash -G docker,www-data appuser
#   -m         创建 home 目录
#   -s         指定 shell
#   -G         附加组
#   -c "desc"  描述
#   -e 2026-12-31  过期时间

# 修改
usermod -aG docker appuser
usermod -L appuser        # 锁定
usermod -U appuser        # 解锁

# 删除
userdel -r appuser        # 连同 home/mail 一起删

# 密码
passwd appuser
passwd -l appuser          # 锁定密码
passwd -u appuser          # 解锁密码
passwd -S appuser          # 查看密码状态

# 密码策略
chage -l appuser           # 查看密码过期策略
chage -M 90 appuser        # 90 天过期
chage -d 0 appuser         # 强制下次修改
chage -W 7 appuser         # 过期前 7 天警告
```

### 用户信息文件

| 文件 | 内容 |
|------|------|
| `/etc/passwd` | 用户名:x:UID:GID:描述:home:shell |
| `/etc/shadow` | 用户名:加密密码:最近改密:最短天数:最长天数:警告期:宽限期:过期:保留 |
| `/etc/group`  | 组名:x:GID:成员列表 |

```bash
# 查看用户信息
id appuser                 # uid/gid/组列表
finger appuser             # 详细登录信息
who                        # 当前登录用户
w                          # 更详细的登录信息
last                       # 最近登录记录
lastlog                    # 所有用户最后登录
```

```bash

groups appuser             # 查看用户所属组
```

## sudo

配置文件：`/etc/sudoers`（必须用 `visudo` 编辑）或 `/etc/sudoers.d/` 下的独立文件。

```bash
visudo -f /etc/sudoers.d/custom
```

```
# 用户级：允许 appuser 免密码重启 nginx
appuser ALL=(ALL) NOPASSWD: /usr/bin/systemctl restart nginx

# 组级：admin 组成员有完全 sudo 权限
%admin ALL=(ALL) ALL

# 命令别名
Cmnd_Alias NETWORK = /sbin/ip, /sbin/ifconfig
%network ALL=(ALL) NOPASSWD: NETWORK

# 限制：只允许特定用户执行特定命令
deploy ALL=(ALL) NOPASSWD: /usr/bin/systemctl *, /bin/systemctl *
deploy ALL=(ALL) !/usr/bin/systemctl *firewall*, !/usr/bin/systemctl *iptables*
```

### sudo 语法格式

```
用户 主机=(可切换哪个用户) 标签: 命令
```

| 字段 | 含义 |
|------|------|
| 用户/组 | 谁可以使用 sudo |
| 主机 | 在哪些机器上生效（通常 ALL） |
| `(ALL)` | 可以切换到哪个用户执行 |
| `NOPASSWD:` | 免密码标签 |
| 命令 | 允许运行的命令（绝对路径） |

## 文件权限

### 基本权限

```bash
chmod 755 file          # rwxr-xr-x
chmod u+x file          # 属主加执行
chmod g-w file          # 组去掉写
chmod o=r file          # 其他人设为只读
chmod a+x file          # 所有人加执行

chmod -R g+w dir/       # 递归组加写
chmod -R u=rwX,go=rX dir/  # X：只有对目录或已有执行权限的文件才加执行

chown user:group file
chown user: file        # 仅改属主
chown :group file       # 仅改组
chown -R user:group dir/
```

### 权限数字速查

```
r=4  w=2  x=1
rwx=7  rw-=6  r-x=5  r--=4  -wx=3  -w-=2  --x=1  ---=0

常见组合：
755  rwxr-xr-x  普通文件/目录
644  rw-r--r--  普通文件
600  rw-------  私钥
700  rwx------  私密脚本目录
```

### 查看文件权限

```bash
ls -l                     # 第 1 列共 10 位
# drwxr-xr-x
# | +-+ +-+ +-+
# |  |   |   + 其他人权限（r-x）
# |  |   + 属组权限（r-x）
# |  + 属主权限（rwx）
# + 文件类型（d=目录 -=文件 l=软链 b=块设备 c=字符设备）
```

## 特殊权限

### SUID / SGID / Sticky

| 权限 | 数字 | 文件效果 | 目录效果 |
|------|------|---------|---------|
| SUID  | 4xxx | 以文件属主身份执行 | 无意义 |
| SGID  | 2xxx | 以文件属组身份执行 | 新建文件继承目录的组 |
| Sticky| 1xxx | 无意义 | 仅文件属主能删除（/tmp） |

```bash
# 设置
chmod u+s file            # SUID
chmod g+s dir/            # SGID
chmod +t /tmp             # 粘滞位

chmod 4755 file           # rwsr-xr-x（SUID）
chmod 2755 dir/           # rwxr-sr-x（SGID）
chmod 1777 /tmp           # rwxrwxrwt（粘滞位）

# 找到所有 SUID
find / -perm -4000 -type f 2>/dev/null
```

### umask

```bash
umask                      # 查看当前掩码（默认 0022）
umask 0027                 # 设置掩码

# 文件默认 666，目录默认 777
# umask 0022 → 文件 644 (666-022)，目录 755 (777-022)
# umask 0027 → 文件 640 (666-027)，目录 750 (777-027)
```

## ACL

当基本权限不够用时（需要对多个用户/组设置不同权限）。

```bash
# 设置
setfacl -m u:alice:rwx file
setfacl -m g:dev-team:rx file
setfacl -m o::- file              # 其他人无权限

# 递归
setfacl -R -m g:dev-team:rx dir/
setfacl -R -m d:g:dev-team:rx dir/  # 默认权限（新文件继承）

# 查看
getfacl file

# 移除
setfacl -b file                   # 移除所有 ACL
setfacl -x u:alice file           # 移除特定用户 ACL

# 备份/恢复
getfacl -R dir/ > acl_backup
setfacl --restore=acl_backup
```

### 权限优先级

```
属主 > ACL 属主 > ACL 具名用户 > 属组 > ACL 具名组 > 其他人
```

## 用户切换

```bash
su - appuser               # 切换用户（加载环境）
su appuser                 # 切换但不加载环境
sudo -u appuser command    # 以某用户身份执行命令
sudo -i -u appuser         # 交互式登录

# sudo 缓存
sudo -k                    # 清除 sudo 缓存
sudo -l                    # 查看当前 sudo 权限
```

## 实践场景

### 场景 1：新建 Web 部署用户

```bash
#!/bin/bash
set -euo pipefail

USER="deploy"
GROUP="www-data"
SERVICE_NAME="myapp"

# 创建用户
id "$USER" 2>/dev/null || useradd -m -s /bin/bash -G "$GROUP" "$USER"

# 创建目录
mkdir -p /opt/$SERVICE_NAME /var/log/$SERVICE_NAME /backup

# 目录权限
chown -R $USER:$GROUP /opt/$SERVICE_NAME /var/log/$SERVICE_NAME /backup

# sudo 限制：只能重启/查看特定服务
cat > /etc/sudoers.d/$USER << EOF
$USER ALL=(ALL) NOPASSWD: /usr/bin/systemctl restart $SERVICE_NAME
$USER ALL=(ALL) NOPASSWD: /usr/bin/systemctl status $SERVICE_NAME
$USER ALL=(ALL) NOPASSWD: /usr/bin/systemctl restart nginx
EOF

visudo -c -f /etc/sudoers.d/$USER

# SSH 密钥
mkdir -p /home/$USER/.ssh
chmod 700 /home/$USER/.ssh
touch /home/$USER/.ssh/authorized_keys
chmod 600 /home/$USER/.ssh/authorized_keys
chown -R $USER:$USER /home/$USER/.ssh

echo "用户创建完成"
```

### 场景 2：SFTP 隔离目录（chroot）

```bash
# 创建用户
useradd -m -s /sbin/nologin sftpuser

# 配置 sshd
cat >> /etc/ssh/sshd_config << EOF

Subsystem sftp internal-sftp

Match User sftpuser
    ChrootDirectory /home/sftpuser
    ForceCommand internal-sftp
    X11Forwarding no
    AllowTcpForwarding no
EOF

# chroot 目录属主必须是 root
chown root:root /home/sftpuser
chmod 755 /home/sftpuser

# 可写目录放在里面
mkdir -p /home/sftpuser/uploads
chown sftpuser:sftpuser /home/sftpuser/uploads

systemctl restart sshd
```

### 场景 3：团队协作目录

```bash
#!/bin/bash
set -euo pipefail

PROJECT_DIR="/data/project"
TEAM_GROUP="dev-team"

# 创建组
groupadd -f "$TEAM_GROUP"

# 创建目录
mkdir -p "$PROJECT_DIR"

# SGID：新文件继承组
chown root:"$TEAM_GROUP" "$PROJECT_DIR"
chmod 2775 "$PROJECT_DIR"   # rwxrwsr-x
# 或
chmod g+s "$PROJECT_DIR"

# ACL：某些成员只读，某些成员读写
# 实习生只读
setfacl -R -m u:intern1:rx "$PROJECT_DIR"
setfacl -m d:u:intern1:rx "$PROJECT_DIR"

# 核心成员读写
setfacl -R -m u:lead1:rwx "$PROJECT_DIR"
setfacl -m d:u:lead1:rwx "$PROJECT_DIR"

# 查看最终权限
getfacl "$PROJECT_DIR"

# umask 统一
echo "umask 002" >> /home/*/.bashrc
```
