# 文本处理三剑客

> 文本处理 | grep/sed/awk | 日志分析、批量替换、数据提取的必备技能

---

## grep —— 文本搜索

```bash
grep -E "error|fail" app.log          # 扩展正则，匹配 error 或 fail
grep -P "\d{3}\.\d{3}\.\d{3}" file    # Perl 正则，更强大
grep -o "[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+" file   # 只输出匹配的 IP
grep -c "ERROR" app.log               # 计数

grep -A 5 "Exception" app.log         # 匹配行 + 后 5 行
grep -B 3 "ERROR" app.log             # 匹配行 + 前 3 行
grep -C 3 "stacktrace" app.log        # 匹配行 + 前后各 3 行

grep -v "^#" config.conf              # 反选，排除注释行
grep -l "TODO" src/*.py               # 只显示包含 TODO 的文件名
grep -r "password" /etc/              # 递归搜索

grep -q "key=value" config.conf && echo "found"   # 静默，用于条件判断
```

### 正则速查

| 元字符 | 含义 | 示例 |
|--------|------|------|
| `^` | 行首 | `^#` 注释行 |
| `$` | 行尾 | `debug$` 以 debug 结尾 |
| `.` | 任意字符 | `f.e` → `fee` `fie` `foe` |
| `*` | 前一个字符重复 0 到任意次 | `fo*` → `f` `fo` `foo` |
| `+` | 前一个字符重复 1 到任意次 | `fo+` → `fo` `foo`（不匹配 `f`） |
| `?` | 前一个字符出现 0 或 1 次 | `colou?r` → `color` `colour` |
| `[]` | 字符集合 | `[Ff]ile` `[0-9]` `[^a-z]` |
| `()` | 分组 | `(error|fail)` |
| `{}` | 重复次数 | `[0-9]{3}` `[0-9]{2,4}` |
| `\|` | 或 | `cat\|dog` |

### 实战：日志上下文提取

```bash
# 找到异常并显示周围 5 行
grep -C 5 "NullPointerException" app.log > error_context.log

# 多个关键词组合 —— 看请求-响应全过程
grep -E "GET /api/order|500|timeout" app.log

# 从一堆日志中提取某时间段内的报错
grep "2026-01-15 1[4-6]:" app.log | grep -E "ERROR|WARN"
```

## sed —— 流编辑器

### 替换

```bash
sed 's/8080/9090/g' file                     # 全局替换
sed 's/8080/9090/2' file                     # 只替换每行第 2 次出现
sed -i 's/8080/9090/g' file                  # 原位修改
sed -i.bak 's/8080/9090/g' file              # 原位修改 + 备份原文件 .bak

sed '2s/old/new/' file                       # 只替换第 2 行
sed '2,5s/old/new/g' file                    # 替换 2-5 行
sed '/^server/s/8080/9090/' file             # 在匹配 ^server 的行上替换

# 分组引用
echo "foo=bar" | sed 's/\([^=]*\)=\(.*\)/\2=\1/'   # bar=foo
```

### 增删

```bash
sed '2a\new line' file                       # 第 2 行后追加
sed '2i\new line' file                       # 第 2 行前插入
sed '2c\replaced line' file                  # 替换第 2 行
sed '2d' file                                # 删第 2 行
sed '/^$/d' file                             # 删空行
sed '/debug/d' file                          # 删包含 debug 的行
sed '2,5d' file                              # 删 2-5 行
```

### 只打印匹配行

```bash
sed -n '/pattern/p' file                     # 只打印匹配行（类似 grep）
sed -n '10,20p' file                         # 只打印 10-20 行
sed -n 'p;n' file                            # 隔一行打一行
```

### 实战：批量修改配置

```bash
# 将 yml 中端口从 8080 改为 9090
sed -i 's/port: 8080/port: 9090/g' config/*.yml

# 注释掉 nginx 中某段
sed -i '/location \/api\//s/^/#/' nginx.conf

# 提取 yaml 中某段落（从 servers: 到下一个顶级键）
sed -n '/^servers:/,/^[a-z]/p' config.yml
```

## awk —— 字段处理语言

```bash
awk '{print $1, $NF}' file                   # 第 1 列、最后一列
awk -F: '{print $1, $3}' /etc/passwd         # 以 : 分隔
awk -F: -v OFS='\t' '{print $1, $6}'         # 输入分隔: 输出分隔\t
```

### 内置变量

| 变量 | 含义 | 示例值 |
|------|------|--------|
| `NR` | 当前行号（全局） | `NR==1` 第一行 |
| `FNR` | 当前文件行号（多文件时分别计数） | `FNR==1` 每个文件的第 1 行 |
| `NF` | 当前行字段数 | `print NF` |
| `$0` | 整行 | `{print $0}` |
| `$1`-`$NF` | 各字段 | `$1`=第一列 |
| `FS` | 输入字段分隔符 | 默认空格/制表符 |
| `OFS` | 输出字段分隔符 | 默认空格 |

### 模式与动作

```bash
awk '/ERROR/' app.log                        # 只打印含 ERROR 的行（类似 grep）

awk '$3 > 500 {print $1, $3}' data           # 第 3 列大于 500 则输出

awk 'NR>1 && NR<10 {print NR, $0}' file      # 打印 2-9 行

awk 'NR%2==0' file                           # 偶数行

awk '/^2026-01-15/,/^2026-01-16/' log        # 时间段区间
```

### BEGIN / END

```bash
awk 'BEGIN {print "=== Report ==="} {sum+=$3} END {print "Sum:", sum}' data
# === Report ===
# Sum: 12345
```

### 数组与统计

```bash
# 统计 IP 出现次数
awk '{count[$1]++} END {for (ip in count) print ip, count[ip]}' access.log \
  | sort -k2 -rn | head -10

# 统计状态码分布
awk '{code[$9]++} END {for (c in code) print c, code[c]}' access.log
```

### printf 格式化

```bash
awk '{printf "%-20s %8d %6.2f%%\n", $1, $2, $3}' data
```

### 内置函数

```bash
awk '{print length($0)}' file                # 每行长度
awk '{print substr($1, 1, 3)}' file          # 取前 3 字符
awk '{gsub(/error/, "ERROR", $0); print}'    # 行内替换
awk '{split($4, arr, ":"); print arr[1]}'    # 按:分割 $4 到数组
```

### 实战：解析 access.log

```bash
# IP 访问 Top 10
awk '{ips[$1]++} END {for (i in ips) print ips[i], i}' access.log \
  | sort -rn | head -10

# 统计某接口小时级调用量
awk '$7 ~ /\/api\/orders/ {hour=substr($4,14,2); h[hour]++}
     END {for (i in h) print i":00", h[i]}' access.log | sort

# 求第 8 列（响应体大小）总和
awk '{sum+=$10} END {print "Total bytes:", sum}' access.log

# 计算平均响应时间（假设第 10 列为耗时 ms）
awk '$10 ~ /^[0-9]+$/ {sum+=$10; n++} END {print "Avg:", int(sum/n)}' access.log
```

## 辅助工具

### sort —— 排序

```bash
sort -n data                                 # 数值排序（默认字典序）
sort -r data                                 # 逆序
sort -k2 -t: file                            # 按第 2 列排序，隔符 :
sort -k3 -rn file                            # 按第 3 列数值逆序
sort -u data                                 # 排序 + 去重
```

### uniq —— 去重与计数

```bash
# ⚠️ uniq 必须配合 sort 使用（只去相邻重复行）
sort file | uniq                             # 去重
sort file | uniq -c                          # 去重 + 计数
sort file | uniq -d                          # 只显示有重复的行
sort file | uniq -u                          # 只显示出现一次的行
```

### cut / tr / wc

```bash
cut -d: -f1,3 /etc/passwd                   # 提取第 1、3 字段
cut -c1-10 file                             # 提取每行前 10 字符

tr 'A-Z' 'a-z' < file                       # 大写转小写
tr -d '\r' < dos.txt > unix.txt             # 删除 \r（DOS→Unix）
tr -s ' ' < file                            # 压缩连续空格为单个

wc -l file                                  # 行数
wc -w file                                  # 单词数
wc -c file                                  # 字节数
```

### tee —— 分流

```bash
# 同时写入文件和标准输出
command | tee log.txt
command | tee -a log.txt                    # 追加模式

# 常用于需要边看边记录的场景
tail -f app.log | tee -a capture.log
```

### paste / join —— 文件拼接

```bash
paste -d, file1 file2                       # 按列拼接，,分隔
paste -d'\t' file1 file2                    # 按列拼接，制表符分隔

join -t: -1 2 -2 1 file1 file2             # 以:分隔，file1 第2列 = file2 第1列 做 join
```

### 典型组合管道

```bash
# 最常用的统计管道
cat access.log | awk '{print $1}' | sort | uniq -c | sort -rn | head -10
```

## 实践场景

### 场景 1：从 access.log 提取状态码 5xx 的 IP 和请求

```bash
awk '$9 ~ /^5[0-9][0-9]$/ {print $1, $7, $9}' access.log \
  | sort | uniq -c | sort -rn | head -20
```

### 场景 2：批量替换所有 yml 中的 8080 → 9090

```bash
# 先确认有哪些文件包含 8080
grep -rl "8080" config/ --include="*.yml"
# 逐个替换并备份
find config/ -name "*.yml" -exec sed -i.bak 's/8080/9090/g' {} \;
```

### 场景 3：统计某接口小时级调用量

```bash
# 假设日志格式：IP - - [10/Jan/2026:13:15:00 +0800] "GET /api/orders HTTP/1.1" 200 1234
awk '$7 == "/api/orders" {
  split($4, dt, "[/:]")
  hour = dt[4]
  h[hour]++
}
END {
  for (i in h) printf "%02d:00  %d\n", i, h[i]
}' access.log | sort
```

### 场景 4：从配置文件中提取并修改值

```bash
# 提取 db 连接信息
sed -n '/^\[database\]/,/^\[/p' config.ini | grep -E "host|port|user|password"

# 批量替换环境变量模板
sed -i "s/\${APP_ENV}/production/g" *.env
sed -i "s/\${LOG_LEVEL}/warn/g" *.env
```

### 场景 5：日志错误聚合报告

```bash
# 提取所有 ERROR 行，统计 TOP 错误类型
grep "ERROR" app.log \
  | sed 's/.*ERROR - //' \
  | awk '{err[$0]++} END {for (e in err) print err[e], e}' \
  | sort -rn | head -10 > top_errors.txt

# 输出示例：
# 235 Connection timeout
# 189 NullPointerException in OrderService.process
# 67  Database connection refused
```
