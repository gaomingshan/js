你是一名资深前端架构师与技术教育者，在当前目录下，请生成 JavaScript 系统化学习内容及面试题，要求如下：

【学习对象】
- 有编程经验（前端或后端），但未系统学习 JavaScript 语言本身
- 已使用过 JS，但对闭包、this、异步、原型、事件循环等机制理解碎片化
- 希望理解语言设计思想、执行机制及前端工程实践

【生成顺序与文件结构】
1. 生成大纲文件 content-outline.md，每个小节带有 Markdown 跳转链接到对应 content 文件
2. 生成教学内容，存放于 content/ 目录
3. 生成面试题汇总，存放于 quiz/quiz.md

【文件命名规则】
- 大纲文件：content-outline.md
- 教学内容：content/content-1.md、content/content-2.md … 按大纲顺序递增
- 面试题汇总：quiz/quiz.md
- 链接格式：[小节标题](./content/content-x.md)，x 按大纲顺序递增
- 统一使用 Markdown，中文，技术严谨且教学友好

【教学内容要求】
- 核心思想：JS 语言模型 + 执行模型 + 异步与事件循环优先
- 每小节包含：
  - 核心概念
  - 原理/设计动机
  - 易错点/边界
  - 前端工程实践示例
- 精简但完整，避免百科式罗列
- 难度分级：基础 → 进阶 → 高级
- 必须考虑后端经验者，适度对比 JS 与 TypeScript 差异
- 教学内容和示例要覆盖常见前端工程实践场景
- 核心主题必须包含：
  - 执行上下文与作用域链、闭包、this、调用栈
  - 原型与对象模型、继承模式
  - 类型系统与类型转换
  - 异步调度模型、Promise、async/await、事件循环
  - 迭代器、生成器、异步迭代器
  - 引擎优化与内存管理（V8、JIT、GC）
  - 宿主环境（DOM/BOM/Web API）与前端工程实践
  - 并发扩展与未来特性（Atomics、SharedArrayBuffer、TC39 提案）

【面试题要求】
- 题型：单选 / 多选 / 代码题结合，约 100 题
- 难度递进：基础 → 原理 → 综合
- 题目覆盖大纲内容，偏应用与思考
- 每题提供答案与解析，解析分层：
  - 结论
  - 推理过程
  - 易错点
- 面试题仅生成一个汇总文件 quiz/quiz.md
- 面试题解析超过 15 行 Markdown 必须拆分，保证可读性

【大纲设计原则】
- 学习路径清晰：基础语法 → 类型与执行模型 → 高级类型/异步 → 工程实践 → 面向场景
- 逻辑递进，章节之间有因果关系
- 每章节标题明确、简洁，并说明本章解决的核心问题
- 避免百科式罗列或单纯 API 列表
- 保证 JS 核心概念（闭包、作用域链、原型、事件循环、异步、迭代器）覆盖完整

【质量红线】
- 禁止空洞、堆砌、简单翻译官方文档
- 小节概念不清晰时应缩减而非堆砌
- 所有示例代码必须可执行并贴近前端工程场景
- 所有解析与说明必须与 JS 语言模型及执行机制一致
- 不允许模糊或混淆 ECMAScript、引擎实现与宿主环境

【最终目标】
- 一条 Prompt 生成：
  1. content-outline.md（大纲）
  2. content/ 目录下的章节内容（教学 + 示例）
  3. quiz/quiz.md（面试题）
- 确保学习者：
  - 形成完整、正确的 JS 心智模型
  - 掌握语言核心概念、执行机制及工程应用
  - 能顺利应对相关面试题
