你是一名资深前端架构师与技术教育者，在 @TypeScript  目录下，请生成 TypeScript 系统化学习内容及面试题，要求如下：

【学习对象】
- 有 JS 基础、想系统学习前端 TS 的开发者（含后端经验者）
- 目标：理解类型系统设计思想、工作原理与工程价值，能在前端工程场景中应用

【生成顺序与文件结构】
1. 生成大纲文件 content-outline.md，每个小节带有 Markdown 跳转链接到对应 content 文件
2. 生成教学内容，存放于 content/ 目录
3. 生成面试题汇总，存放于 quiz/quiz.md 

【文件命名规则】
- 大纲文件：content-outline.md 
- 教学内容：content/content-1.md、content/content-2.md … 按大纲顺序递增
- 面试题汇总：quiz/quiz.md 
- 链接格式：[小节标题](./content/content-x.md)，x 按大纲顺序递增
- 统一使用 Markdown，中文，技术严谨且教学友好

【教学内容要求】
- 核心思想：结构类型系统、类型与运行时分离、类型推断优先
- 每小节包含：核心概念 + 原理/设计动机 + 易错点/边界 + 前端工程实践示例
- 精简但完整，避免百科式罗列
- 难度分级：基础 → 进阶 → 高级

【面试题要求】
- 单选 / 多选 / 代码题结合，约 100 题
- 难度递进：基础 → 原理 → 综合
- 题目覆盖大纲内容，偏应用与思考
- 每题提供答案与解析，解析分层：结论、推理过程、易错点，保证 Markdown 可读性
- 面试题仅生成一个汇总文件 quiz/quiz.md 

【质量红线】
- 禁止空洞、堆砌、简单翻译官方文档
- 如果小节内容概念不清晰，应缩减而非堆砌
- 面试题解析超过 15 行 Markdown 必须拆分，保证可读性

【额外说明】
- 每小节和面试题都考虑后端经验者，可适度对比 JS 与 TS 差异
- 教学内容和题目应覆盖常见前端工程实践场景
- 大纲文件必须生成 Markdown 链接，格式为 [小节标题](./content/content-x.md)，确保点击可跳转
